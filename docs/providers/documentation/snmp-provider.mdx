## Overview

The SNMP provider is a push-based consumer that listens for incoming SNMP trap PDUs on a configurable UDP port. It decodes both SNMPv1 Trap-PDU and SNMPv2c trap formats, maps well-known trap types to alert severities, and forwards them to Keep as alerts.

Supported trap types and their default severity mapping:

| Trap Type             | Severity |
|-----------------------|----------|
| coldStart             | Warning  |
| warmStart             | Info     |
| linkDown              | High     |
| linkUp                | Info     |
| authenticationFailure | Warning  |
| egpNeighborLoss       | High     |
| Enterprise-specific   | Warning  |

## Connecting with the Provider

1. Decide which UDP port to listen on. The standard SNMP trap port is `162` (requires root). Use `1162` or another unprivileged port if running without elevated privileges.
2. Set the community string to match the devices that will send traps.
3. Configure the provider in Keep with the port, bind address, and community string.
4. Point your SNMP-capable devices (routers, switches, servers) to send traps to the Keep host on the configured port.
