## Connecting with the Provider

1. Add the SNMP provider and configure the UDP listener address, port, and accepted community string.
2. Make sure the Keep backend can receive UDP traffic on the configured port.
3. Configure your network devices or SNMP trap forwarder to send SNMP v1 or v2c traps to that address.

The default SNMP trap port is `162`, which usually requires elevated privileges. For local development, use an unprivileged port such as `1162`.

Example `snmptrap` command:

```bash
snmptrap -v 2c -c public 127.0.0.1:1162 '' \
  1.3.6.1.6.3.1.1.5.3 \
  1.3.6.1.2.1.2.2.1.2 s eth0
```

## Notes

- The provider uses a native UDP listener and does not require an external SNMP dependency.
- Incoming traps are decoded on the UDP listener thread and queued before being pushed to Keep, so Keep API latency does not block packet reception.
- The provider accepts SNMP v1 and v2c traps. SNMPv3 authentication and encryption are not supported yet.
