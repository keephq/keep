## Overview

The SNMP Provider connects Keep to the SNMP (Simple Network Management Protocol) ecosystem â€” the universal standard for network device monitoring.
Virtually every router, switch, firewall, server, UPS, and printer speaks SNMP, making this provider essential for on-premises and hybrid infrastructure monitoring.

Two integration modes are supported:

- **Push mode (trap receiver)** â€” Keep starts a built-in UDP listener that receives SNMP traps directly from devices, or an external trap daemon (snmptrapd) forwards trap data to Keep as JSON via HTTP webhook.
- **Pull mode (polling)** â€” Keep periodically queries SNMP OIDs on target devices to surface threshold violations as alerts.

Both modes support **SNMPv1**, **SNMPv2c**, and **SNMPv3** (with authentication and privacy).

## Authentication

| Field | Required | Description |
|-------|----------|-------------|
| `host` | No | Bind address for the trap listener (default: `0.0.0.0`) |
| `port` | No | UDP port for traps (default: `162`; use `1620+` for non-root) |
| `community_string` | No | SNMPv1/v2c community string (default: `public`) |
| `version` | No | SNMP version: `1`, `2c`, or `3` (default: `2c`) |
| `username` | No* | SNMPv3 username (required for v3) |
| `auth_key` | No | SNMPv3 authentication passphrase |
| `auth_protocol` | No | SNMPv3 auth protocol: `MD5` or `SHA` (default: `SHA`) |
| `priv_key` | No | SNMPv3 privacy passphrase |
| `priv_protocol` | No | SNMPv3 privacy protocol: `DES` or `AES` (default: `AES`) |
| `oids_mapping` | No | JSON map of OID prefixes â†’ custom severity/name overrides |
| `poll_enabled` | No | Enable periodic SNMP polling (default: `false`) |
| `poll_targets` | No | JSON list of `{"host": "...", "oids": [...]}` polling targets |
| `poll_interval` | No | Polling interval in seconds (default: `60`) |

\* Required when `version` is `3`.

## Push Mode â€” Receiving Traps

### Option A: Built-in UDP Listener

Keep starts a UDP trap listener on `host:port` automatically.
Configure your devices to send traps to Keep's IP:

```
# Cisco IOS example
snmp-server host <keep-ip> traps version 2c public

# SNMPv3 (authenticated + encrypted)
snmp-server host <keep-ip> traps version 3 priv <username>
```

### Option B: snmptrapd JSON Forwarding

Use Net-SNMP's `snmptrapd` as a trap receiver and forward to Keep's webhook:

1. Edit `/etc/snmp/snmptrapd.conf`:
```
traphandle default /usr/bin/curl -s -X POST \
  -H 'Content-Type: application/json' \
  -d @- {keep_webhook_api_url}
```

2. Restart snmptrapd:
```bash
systemctl restart snmptrapd
```

### Webhook Payload Format

Keep accepts two JSON formats:

**Single trap:**
```json
{
  "trap_oid": "1.3.6.1.6.3.1.1.5.3",
  "source": "192.168.1.1",
  "version": "2c",
  "community": "public",
  "varbinds": [
    {"oid": "1.3.6.1.2.1.2.2.1.1.1", "value": "1"}
  ]
}
```

**Batch (multiple traps):**
```json
{
  "traps": [
    {"trap_oid": "...", "source": "...", "varbinds": [...]},
    {"trap_oid": "...", "source": "...", "varbinds": [...]}
  ]
}
```

## Pull Mode â€” Polling OIDs

Enable polling to periodically query devices:

```json
{
  "poll_enabled": true,
  "poll_interval": 60,
  "poll_targets": [
    {
      "host": "192.168.1.1",
      "oids": [
        "1.3.6.1.2.1.1.1.0",
        "1.3.6.1.2.1.2.2.1.8.1"
      ]
    }
  ]
}
```

## OID Mapping

Customise severity and alert names for specific OID prefixes:

```json
{
  "1.3.6.1.4.1.9": {"severity": "critical", "name": "Cisco Device Alert"},
  "1.3.6.1.6.3.1.1.5.3": {"severity": "critical", "name": "Interface Down"}
}
```

Longest-prefix matching is used. Built-in OID mappings (RFC-1907, IF-MIB, BGP4, UPS-MIB) are included by default.
