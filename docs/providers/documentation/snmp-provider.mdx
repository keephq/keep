## Overview

The Simple Network Management Protocol (SNMP) is a widely used standard for monitoring and managing network devices such as routers, switches, firewalls, and servers.

The SNMP Provider in Keep acts as a trap receiver, listening for incoming SNMP notifications and converting them into alerts in real time.

---

## Features

### Trap Ingestion
- Receives SNMP traps over UDP
- Supports SNMPv1, SNMPv2c, and SNMPv3

### Alert Generation
Each SNMP trap becomes a Keep alert with:

- Title derived from trap OID
- Full description of OID-value pairs
- Severity mapping or auto-detection
- Source: snmp
- Raw JSON trap payload

---

## Severity Mapping Logic

Priority order:

1. Custom mapping (highest priority)
   - User-defined JSON mapping of OIDs or values

2. Built-in OID mapping
   - Example: linkDown, UPS failure, authenticationFailure

3. Keyword detection
   - critical / fail / down → CRITICAL
   - error / major / high → HIGH
   - warning / degraded → WARNING
   - ok / up / restored → INFO

4. Default fallback → WARNING

---

## Inputs

- SNMP trap packets via UDP
- From routers, switches, servers, monitoring agents

---

## Outputs

Generated alert contains:

- title
- description
- severity
- source = snmp
- raw_data (JSON)

## Authentication Parameters

### Common (SNMPv1 / SNMPv2c)

| Parameter        | Required | Description              | Default |
|----------------|----------|--------------------------|---------|
| listen_address | yes       | IP to bind receiver      | 0.0.0.0 |
| port           | No       | UDP port                 | 1162     |
| community      | No       | SNMP community string    | public  |
| severity_mapping | No     | JSON mapping rules      | null    |

---

### SNMPv3

| Parameter     | Required | Description              |
|--------------|----------|--------------------------|
| username     | Yes      | SNMPv3 username          |
| auth_protocol| No       | MD5 or SHA               |
| auth_key     | If auth set | Authentication password |
| priv_protocol| No       | DES or AES               |
| priv_key     | If priv set | Encryption password     |

---

## Connecting with Provider

### SNMPv2c Configuration

```providers:
  snmp:
    type: snmp
    authentication:
      listen_address: 0.0.0.0
      port: 1162
      community: public 
```

---

### Severity Mapping Example

```providers:
  snmp:
    type: snmp
    authentication:
      severity_mapping: |
        {
          "1.3.6.1.6.3.1.1.5.3": "CRITICAL",
          "authenticationFailure": "WARNING"
        }
```

---

### SNMPv3 Configuration

```providers:
  snmp:
    type: snmp
    authentication:
      listen_address: 0.0.0.0
      port: 1162
      username: snmpuser
      auth_protocol: SHA
      auth_key: authpass
      priv_protocol: AES
      priv_key: privpass

```

## Docker Testing Setup (SNMP Trap Sender)

This setup runs a lightweight container to generate SNMP traps and send them to your Keep SNMP provider.

---

### 1. Create Docker Compose file

Create a file named `docker-compose.yml`

`version: "3.8"

services:
  snmp-tools:
    image: debian:bullseye
    container_name: snmp-tools
    command: tail -f /dev/null
    tty: true
    stdin_open: true`

---

### 2. Start the container

`docker-compose up -d`

Verify it is running:

`docker ps`

---

### 3. Install SNMP tools inside container

`docker exec -it snmp-tools bash`

Inside the container run:

`apt-get update
apt-get install -y snmp`

Exit container:

`exit`

---

### 4. Send SNMP Trap to Keep Provider

### If Keep is running on your host machine:

Use:

`docker exec snmp-tools snmptrap -v 2c -c public host.docker.internal:1162 '' \
1.3.6.1.6.3.1.1.5.3 \
1.3.6.1.2.1.2.2.1.1.2 i 2`

---

### If running on Linux (no host.docker.internal):

Get your host IP:

`ip addr | grep inet`

Then run:

`docker exec snmp-tools snmptrap -v 2c -c public <HOST-IP>:1162 '' \
1.3.6.1.6.3.1.1.5.3 \
1.3.6.1.2.1.2.2.1.1.2 i 2`

---

## 6. Stop and cleanup

`docker-compose down`

## Notes

- Supports SNMPv1, SNMPv2c, SNMPv3
- Port 1162 requires elevated privileges
- Use ports >1024 for testing (e.g. 101162)
- SNMPv3 engine IDs are auto-handled
