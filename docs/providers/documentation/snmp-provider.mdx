## Connecting with the Provider

1. Pick a UDP port for the trap listener. Port 162 is the SNMP default but
   requires elevated privileges; for local testing or container deployments a
   high port (e.g. 1162) is easier.
2. Configure your network devices to forward traps to the host running Keep
   on the chosen port using the configured community string.
3. Add the SNMP provider in Keep with the matching `listen_host`,
   `listen_port`, `community_string`, and `snmp_version`.

## Mapping

Each received trap is converted to a Keep alert as follows:

- `name` and `fingerprint` -> the trap OID (`snmpTrapOID.0`)
- `description` -> a newline-joined dump of every variable binding
- `severity` -> heuristic match against well-known keywords
  (`critical`, `error`, `warning`, `info`, `debug`)
- `labels.trap_oid`, `labels.snmp_version`, `labels.source_address`,
  plus every var bind mirrored as its own label
