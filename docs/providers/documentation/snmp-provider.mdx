## Overview

The SNMP provider enables Keep to receive SNMP trap notifications forwarded by an external SNMP trap manager. Because Keep does not run an SNMP listener itself, you use a tool such as **snmptrapd**, **Net-SNMP**, **Zabbix**, or **Nagios** to receive raw traps and then forward them as JSON to Keep's webhook endpoint.

This is the same zero-external-dependency pattern used by the Checkmk, Wazuh, and Netdata providers.

## How it works

```
Network device  →  SNMP trap (UDP 162)  →  snmptrapd / Net-SNMP
                                                  ↓
                                       HTTP POST JSON payload
                                                  ↓
                                       Keep webhook endpoint
```

## Expected JSON payload

Keep's SNMP provider parses the following fields from the incoming JSON body:

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `oid` | string | recommended | Trap OID (e.g. `1.3.6.1.6.3.1.1.5.3`) |
| `host` | string | no | Source host IP or hostname |
| `message` | string | no | Human-readable description |
| `severity` | string | no | `critical`, `high`, `warning`, `info`, or `low` |
| `uptime` | string | no | sysUpTime from the trap PDU |
| `variables` | object | no | Varbind dictionary `{ oid: value }` |
| `id` | string | no | Deduplication ID |

Example payload:

```json
{
  "oid": "1.3.6.1.6.3.1.1.5.3",
  "host": "192.168.1.10",
  "message": "linkDown on interface eth0",
  "severity": "high",
  "uptime": "3:14:15.00",
  "variables": {
    "1.3.6.1.2.1.2.2.1.8.2": "2"
  }
}
```

## OID-to-severity defaults

If no `severity` field is present, Keep maps well-known OIDs automatically:

| OID | Trap name | Default severity |
|-----|-----------|-----------------|
| `1.3.6.1.6.3.1.1.5.1` | coldStart | LOW |
| `1.3.6.1.6.3.1.1.5.2` | warmStart | LOW |
| `1.3.6.1.6.3.1.1.5.3` | linkDown | HIGH |
| `1.3.6.1.6.3.1.1.5.4` | linkUp | INFO |
| `1.3.6.1.6.3.1.1.5.5` | authenticationFailure | WARNING |
| Other / enterprise | — | INFO |

## Connecting SNMP to Keep

### 1. Add the SNMP provider in Keep

In the Keep UI, go to **Providers → Add Provider → SNMP**. No credentials are needed — just save it to get your webhook URL:

```
https://api.keephq.dev/alerts/event/snmp
```

### 2. Configure snmptrapd to forward traps as JSON

Install Net-SNMP and create a handler script that converts trap data to JSON and POSTs it to Keep.

Create `/usr/local/bin/snmp-to-keep.sh`:

```bash
#!/usr/bin/env bash
# Called by snmptrapd with trap data on stdin.
# Usage: snmp-to-keep.sh <hostname> <ip-address>

KEEP_URL="https://api.keephq.dev/alerts/event/snmp"
KEEP_API_KEY="<your-keep-api-key>"

HOST="$1"
IP="$2"

# Read varbinds from stdin
VARBINDS=""
while IFS= read -r line; do
    VARBINDS="$VARBINDS $line"
done

# Build a minimal JSON payload; extend as needed
PAYLOAD=$(printf '{"host":"%s","message":"SNMP trap from %s","variables":{}}' "$HOST" "$IP")

curl -s -X POST "$KEEP_URL" \
  -H "Content-Type: application/json" \
  -H "X-API-KEY: $KEEP_API_KEY" \
  -d "$PAYLOAD"
```

Make it executable:

```bash
chmod +x /usr/local/bin/snmp-to-keep.sh
```

Add to `/etc/snmp/snmptrapd.conf`:

```
authCommunity execute public
traphandle default /usr/local/bin/snmp-to-keep.sh
```

Restart snmptrapd:

```bash
systemctl restart snmptrapd
```

### 3. Using Zabbix as the trap forwarder

In Zabbix, create a **Media type** of type *Webhook* that POSTs to the Keep webhook URL with the JSON structure shown above. Map `{EVENT.NAME}` to `message`, `{HOST.IP}` to `host`, and the Zabbix severity to Keep's `severity` field.

### 4. Test the integration

Send a test trap from any host:

```bash
snmptrap -v 2c -c public <your-snmptrapd-host> '' 1.3.6.1.6.3.1.1.5.3
```

You should see a new alert appear in Keep within seconds.
