## Overview

The SNMP Provider enables integration between Keep and any SNMP-capable network device (routers, switches, firewalls, UPS units, servers, etc.). SNMP traps are forwarded to Keep's webhook endpoint as JSON, where they are automatically mapped to Keep alerts with appropriate severity and status.

This is a **webhook-only** provider — no outbound connections or API polling. Your SNMP trap daemon (e.g. Net-SNMP's `snmptrapd`) sends trap notifications to Keep as HTTP POST requests.

## Supported Trap Types

The provider automatically recognizes standard SNMP trap types:

| Trap OID | Name | Default Severity | Default Status |
|----------|------|-----------------|----------------|
| 1.3.6.1.6.3.1.1.5.1 | coldStart | INFO | FIRING |
| 1.3.6.1.6.3.1.1.5.2 | warmStart | INFO | FIRING |
| 1.3.6.1.6.3.1.1.5.3 | linkDown | CRITICAL | FIRING |
| 1.3.6.1.6.3.1.1.5.4 | linkUp | INFO | RESOLVED |
| 1.3.6.1.6.3.1.1.5.5 | authenticationFailure | WARNING | FIRING |
| 1.3.6.1.6.3.1.1.5.6 | egpNeighborLoss | WARNING | FIRING |

Enterprise-specific traps (generic-trap = 6) are also supported — you can pass an explicit `severity` field in the JSON payload to override the default.

## Expected JSON Payload

```json
{
    "host": "router1.example.com",
    "source_ip": "192.168.1.1",
    "trap_oid": "1.3.6.1.6.3.1.1.5.3",
    "generic_trap": 2,
    "version": "v2c",
    "community": "public",
    "message": "Interface GigabitEthernet0/1 went down",
    "timestamp": "2026-03-01T10:30:00Z",
    "variables": {
        "1.3.6.1.2.1.2.2.1.2": "GigabitEthernet0/1",
        "1.3.6.1.2.1.2.2.1.8": "2"
    }
}
```
