## Overview

The SNMP provider lets you receive SNMP trap notifications in Keep. It works in push mode: a trap receiver (snmptrapd, snmptt, or similar) converts incoming SNMP traps to JSON and forwards them to Keep via HTTP webhook.

This covers the standard SNMPv1 generic trap types (coldStart, warmStart, linkDown, linkUp, authenticationFailure, egpNeighborLoss) and enterprise-specific traps. Severity and status are derived automatically from the trap type and payload fields.

## Connecting SNMP to Keep

### Receiving traps via webhook

You need an SNMP trap receiver that can translate traps into JSON HTTP POST requests. Two common options:

#### Option A: snmptrapd with a webhook script

1. Install `snmptrapd` (part of Net-SNMP).
2. Write a handler script that converts trap data to JSON and POSTs to:

```
{keep_webhook_api_url}
```

3. Include the header `X-API-KEY` with your Keep API key.

4. Example JSON payload:

```json
{
  "trap_oid": "1.3.6.1.6.3.1.1.5.3",
  "generic_trap": 2,
  "agent_address": "192.168.1.1",
  "community": "public",
  "description": "Interface eth0 is down",
  "varbinds": {
    "1.3.6.1.2.1.2.2.1.1": "2",
    "1.3.6.1.2.1.2.2.1.7": "2"
  },
  "timestamp": "2025-01-15T10:30:00Z"
}
```

#### Option B: snmptt (SNMP Trap Translator)

1. Install and configure snmptt.
2. In the handler definition, use the EXEC directive to run a curl command:

```bash
EXEC curl -X POST {keep_webhook_api_url} \
  -H "Content-Type: application/json" \
  -H "X-API-KEY: YOUR_KEEP_API_KEY" \
  -d '{"trap_oid":"$O","agent_address":"$A","description":"$Fz","community":"$C","severity":"$s","timestamp":"$x"}'
```

### Supported payload fields

| Field | Description |
|---|---|
| `trap_oid` / `snmpTrapOID` | The OID of the trap |
| `generic_trap` | SNMPv1 generic trap number (0-6) |
| `specific_trap` | Enterprise-specific trap number |
| `agent_address` | IP of the device sending the trap |
| `community` | SNMP community string |
| `enterprise` | Enterprise OID |
| `description` / `message` | Human-readable trap description |
| `severity` | Explicit severity (critical, warning, info, etc.) |
| `status` | Explicit status (firing, resolved, etc.) |
| `varbinds` | Variable bindings as key-value pairs |
| `timestamp` | Trap timestamp (ISO 8601 or UNIX epoch) |
