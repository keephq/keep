## Overview

The SNMP provider enables you to receive SNMP trap events as Keep alerts. It works as a
**webhook receiver**: an external SNMP trap receiver (such as **snmptrapd**, **SNMPTT**, or
network management systems like Zabbix or Nagios) listens for traps on UDP port 162 and
forwards them to Keep's webhook URL as JSON.

Keep automatically resolves alert severity and status from the trap OID or explicit fields:

| Trap | Severity | Status |
|---|---|---|
| `coldStart` | Critical | Firing |
| `warmStart` | Warning | Resolved |
| `linkDown` | High | Firing |
| `linkUp` | Info | Resolved |
| `authenticationFailure` | Warning | Firing |
| `egpNeighborLoss` | High | Firing |
| Enterprise-specific | Auto-detected | Firing |

## Connecting SNMP to Keep

### Option 1 — snmptrapd with a forwarding script (recommended)

1. **Install net-snmp** on the trap receiver host:

```bash
sudo apt-get install snmptrapd   # Debian/Ubuntu
sudo yum install net-snmp        # RHEL/CentOS
```

2. **Configure snmptrapd** (`/etc/snmp/snmptrapd.conf`):

```conf
# Accept traps from the "public" community and forward via the handler script
authCommunity log,execute,net public
traphandle default /usr/local/bin/keep-snmp-forward
```

3. **Create the handler script** `/usr/local/bin/keep-snmp-forward`:

```bash
#!/bin/bash
# snmptrapd passes: $1=hostname, $2=source_ip, then trap data on stdin
KEEP_URL="https://api.keephq.dev/alerts/event/snmp?api_key=YOUR_API_KEY"

# Read all varbind lines from stdin
VARS=$(cat)

curl -s -X POST "$KEEP_URL" \
  -H "Content-Type: application/json" \
  -d "{
    \"source_ip\": \"$2\",
    \"community\": \"public\",
    \"description\": $(echo "$VARS" | python3 -c 'import json,sys; print(json.dumps(sys.stdin.read()))')
  }"
```

4. **Make it executable** and restart snmptrapd:

```bash
chmod +x /usr/local/bin/keep-snmp-forward
sudo systemctl restart snmptrapd
```

### Option 2 — Direct JSON POST

Any system that can issue HTTP requests can forward traps directly to Keep.
Use `{keep_webhook_api_url}` as the POST target.

**Recommended JSON payload**:

```json
{
  "source_ip": "192.168.1.10",
  "community": "public",
  "trap_oid": "1.3.6.1.6.3.1.1.5.3",
  "trap_name": "linkDown",
  "uptime": "567890",
  "varbinds": {
    "ifIndex": "2",
    "ifAdminStatus": "1",
    "ifOperStatus": "2"
  }
}
```

**Flat JSON** (simpler forwarders):

```json
{
  "oid": "1.3.6.1.4.1.9.9.43.2.0.1",
  "message": "Configuration change on router",
  "source": "172.16.0.1",
  "severity": "warning"
}
```

### Option 3 — Zabbix SNMP media type

Configure a Zabbix **Script** media type that POSTs to the Keep webhook URL.
Use the JSON payload format above with the relevant Zabbix macros.

## Supported JSON fields

| Field | Description |
|---|---|
| `source_ip` / `src` / `agent` / `host` | IP address of the device that sent the trap |
| `trap_oid` / `oid` / `enterprise` | SNMP OID of the trap |
| `trap_name` | Human-readable name (resolved automatically from OID if absent) |
| `community` | SNMP community string |
| `uptime` | Agent uptime (sysUpTime) at the time of the trap |
| `varbinds` | Dict of OID → value variable bindings |
| `description` / `message` / `output` | Alert description |
| `severity` / `priority` | Override Keep severity (`critical`, `high`, `warning`, `info`, `low`) |
| `status` | Override Keep status (`firing`, `resolved`, `ok`) |
| `timestamp` | ISO-8601 timestamp (defaults to current time) |
| `id` | Alert ID (auto-generated if absent) |
