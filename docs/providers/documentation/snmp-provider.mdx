## Connecting with the Provider

1. Choose a UDP port for traps. Port **162** is standard but often requires elevated privileges; **9162** is a common choice for development.
2. Set **community** to match the SNMPv1/v2c community string configured on your network devices or trap forwarders.
3. Install the provider in Keep. When the API starts, the listener runs in a background thread (same pattern as the Kafka consumer).
4. Point your devices or an SNMP trap forwarder at `host:port` for this Keep instance.

SNMP **v3** (USM) is not configured in this version; use a forwarder to translate v3 to v2c if needed.
