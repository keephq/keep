## Overview

The SonarQube provider integrates with [SonarQube](https://www.sonarsource.com/products/sonarqube/) and [SonarCloud](https://sonarcloud.io/) to surface code quality alerts directly in Keep. It can pull open issues (bugs, vulnerabilities, code smells) and receive webhook notifications when a Quality Gate status changes after a project analysis.

## Connecting with the Provider

### Authentication

SonarQube uses token-based authentication.

1. Log in to your SonarQube or SonarCloud instance.
2. Go to **Your Profile** → **Security** → **Generate Tokens**.
3. Create a new **User Token** and copy it.
4. Provide the token along with your SonarQube server URL when configuring the provider.

For **SonarCloud**, also provide your **Organization Key** (visible in your SonarCloud account settings).

### Webhook Setup (optional)

To receive real-time alerts when a Quality Gate status changes:

1. Go to your SonarQube project → **Project Settings** → **Webhooks**.
2. Create a new webhook pointing to your Keep webhook endpoint.
3. SonarQube will send a POST request to Keep after each analysis.

See [SonarQube Webhook Documentation](https://docs.sonarsource.com/sonarqube/latest/project-administration/webhooks/) for details.

## Inputs

The `_get_alerts` method fetches issues with the following default filters:

- **Statuses**: `OPEN`, `CONFIRMED`, `REOPENED`
- **Types**: `BUG`, `VULNERABILITY`, `CODE_SMELL`

The `_format_alert` method processes Quality Gate webhook payloads and reports:

- **status** (`ERROR`, `WARN`, or `OK`): mapped to FIRING or RESOLVED
- **failedConditions**: threshold breaches for metrics like coverage, duplication, etc.

## Severity Mapping

| SonarQube Severity | Keep Severity |
|--------------------|---------------|
| BLOCKER            | Critical      |
| CRITICAL           | High          |
| MAJOR              | High          |
| MINOR              | Warning       |
| INFO               | Info          |
