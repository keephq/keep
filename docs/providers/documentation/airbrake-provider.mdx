## Overview

[Airbrake](https://airbrake.io/) is an error and performance monitoring platform that captures exceptions, crashes, and performance issues from web applications across multiple languages including Ruby, Python, JavaScript, Go, Java, and more. Airbrake helps engineering teams quickly identify, triage, and resolve errors in production.

## Connecting Airbrake to Keep

Airbrake can be connected to Keep in two ways:

1. **Webhook** — Airbrake pushes alerts to Keep in real time when new errors occur.
2. **API Polling** — Keep periodically polls the Airbrake API to fetch open error groups.

### Option 1: Webhook Integration

### Step 1: Set Up the Webhook in Airbrake

1. Log in to your [Airbrake dashboard](https://airbrake.io).
2. Navigate to your project and click **Settings** → **Integrations**.
3. Select **Webhook** from the integrations list.
4. Enter your Keep webhook URL:

   ```
   {keep_webhook_api_url}
   ```

5. Add a custom HTTP header for authentication:
   - **Header**: `X-API-KEY: {api_key}`

6. Click **Save** to activate the webhook.

Airbrake will now push new and resolved errors to Keep in real time.

### Option 2: API Polling

To enable API polling, configure the Airbrake provider with your **Project ID** and **Project Key**:

1. In your Airbrake project, go to **Settings** → **Project Settings**.
2. Copy the **Project ID** and **Project Key** (API key).
3. Add the Airbrake provider in Keep with these credentials.

Keep will periodically fetch open error groups from Airbrake.

## Alert Payload

Airbrake sends webhook payloads in JSON format. The payload is wrapped in an `error` object:

```json
{
  "error": {
    "id": 12345,
    "error_class": "NoMethodError",
    "error_message": "undefined method 'foo' for nil:NilClass",
    "environment": "production",
    "action": "create",
    "url": "https://airbrake.io/projects/...",
    "created_at": "2024-01-01T00:00:00Z"
  }
}
```

Keep maps Airbrake fields as follows:

| Airbrake Field | Keep Field |
|---|---|
| `id` | Alert ID |
| `error_class` / `type` | Alert name |
| `error_message` / `message` | Description |
| `action` | Alert status (create → FIRING, resolve → RESOLVED) |
| `environment` | Environment |
| `url` / `airbrake_url` | Alert URL |
| `created_at` | Last received time |
