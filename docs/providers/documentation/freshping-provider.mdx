## Overview

[Freshping](https://www.freshping.io/) is an uptime monitoring service by Freshworks. It checks the availability of your websites and APIs from multiple global locations and alerts you when a check goes down or recovers.

The Keep Freshping provider receives real-time webhook notifications when Freshping checks become unavailable or recover, surfacing them as alerts in Keep.

## Connecting Freshping to Keep

### Setting up a webhook alert contact

1. Log in to [freshping.io](https://www.freshping.io/).
2. In the left sidebar, click **Alerts** → **Alert Contacts**.
3. Click **New Alert Contact** in the top right.
4. Select **Webhook** as the contact type.
5. Enter your Keep webhook URL as the endpoint:
   ```
   https://api.keephq.dev/alerts/event/freshping
   ```
6. Under **Custom Headers**, add:
   - Key: `X-API-KEY`
   - Value: your Keep API key (Keep → Settings → API Keys → Create)
7. Optionally, set a **Secret Token** in Freshping. Store the same token as `webhook_secret` when configuring the Freshping provider in Keep.
8. Click **Save**.

### Assigning the alert contact to checks

1. In Freshping, go to **Checks** and open the check you want to monitor.
2. Scroll to **Alert Contacts** and click **+ Add Alert Contact**.
3. Select the webhook contact you created and save.

Freshping will now send webhook payloads to Keep when the check status changes.

## Alert fields

| Keep field | Freshping source |
|---|---|
| `id` | `check_id` + `report_generated_at` |
| `name` | `check_name` |
| `description` | `summary` |
| `severity` | Mapped from `check_status`: unavailable → CRITICAL, unknown → WARNING |
| `status` | Mapped from `check_status`: unavailable → FIRING, available → RESOLVED |
| `lastReceived` | `report_generated_at` |
| `url` | `check_url` |
