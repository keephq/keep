## Overview

The SkyWalking provider enables integration between Keep and [Apache SkyWalking](https://skywalking.apache.org/), an open-source APM and observability platform for distributed systems. It allows you to receive SkyWalking alarm notifications in Keep via webhooks.

## Connecting SkyWalking to Keep

To connect SkyWalking to Keep, you need to configure a webhook hook in the SkyWalking OAP server's alarm settings.

### Step 1: Configure the Webhook

Open your SkyWalking backend configuration file `config/alarm-settings.yml` and add a webhook section:

```yaml
webhook:
  keep:
    is-default: true
    urls:
      - {keep_webhook_api_url}
    headers:
      X-API-KEY: {api_key}
```

### Step 2: Restart SkyWalking OAP

Restart the SkyWalking OAP server for the changes to take effect.

### Step 3: Verify

Trigger a test alarm in SkyWalking (e.g., by creating a rule with a low threshold). You should see the alert appear in Keep.

## Severity Mapping

SkyWalking alarms can include a `level` tag to indicate severity. The mapping is:

| SkyWalking Tag Value | Keep Severity |
|---------------------|---------------|
| CRITICAL / CRIT     | Critical      |
| HIGH                | High          |
| WARNING / WARN      | Warning       |
| LOW                 | Low           |
| INFO / OK           | Info          |

If no level tag is present, alerts default to **Warning** severity.

## Status Mapping

| SkyWalking State     | Keep Status |
|---------------------|-------------|
| recoveryTime = null  | Firing      |
| recoveryTime is set  | Resolved    |

## Supported Scopes

SkyWalking can send alarms for the following entity scopes:

- **Service** — Service-level metrics alerts
- **ServiceInstance** — Instance-level metrics alerts
- **Endpoint** — Endpoint-level metrics alerts
- **ServiceRelation** — Service-to-service relationship alerts
- **ServiceInstanceRelation** — Instance-to-instance relationship alerts
- **EndpointRelation** — Endpoint-to-endpoint relationship alerts
