## Overview

[Apache SkyWalking](https://skywalking.apache.org/) is an open-source APM (Application Performance Monitoring) and observability platform. It provides distributed tracing, service mesh telemetry analysis, metric aggregation, and alarm functionality for microservices, cloud-native, and container-based architectures.

The Keep SkyWalking provider supports:

| Mode | Description |
|------|-------------|
| **Pull** | Queries active alarms from the SkyWalking OAP server via the GraphQL API |
| **Webhook** | Receives real-time alarm events pushed by SkyWalking's webhook hook |

## Prerequisites

- SkyWalking OAP server **v8.0+** (GraphQL API available since v6)
- Network access from Keep to the SkyWalking OAP server on its GraphQL port (default: `12800`)

## Authentication

SkyWalking OAP does not require authentication by default. If your deployment has token-based authentication enabled, provide the bearer token.

| Field | Required | Description |
|-------|----------|-------------|
| `host` | ✅ | OAP server URL, e.g. `http://skywalking-oap:12800` |
| `token` | ✗ | Bearer token (only if auth is configured on the OAP server) |

## Connecting SkyWalking to Keep

### Pull Mode (GraphQL)

1. Open Keep → **Providers** → **Connect Provider** → **Apache SkyWalking**.
2. Enter the `host` of your SkyWalking OAP server (e.g. `http://oap-server:12800`).
3. Optionally enter a `token` if your instance requires authentication.
4. Click **Save**. Keep will poll for active alarms every configured interval.

### Webhook Mode (Push)

SkyWalking can push alarm notifications to an HTTP endpoint as soon as they fire.

1. Edit your SkyWalking `alarm-settings.yml`:

```yaml
hooks:
  webhook:
    default:
      is-default: true
      urls:
        - https://<your-keep-instance>/alerts/event/skywalking
```

2. Restart the OAP server to apply the configuration.
3. SkyWalking will POST alarm payloads to the Keep endpoint in JSON format.

## Alert Schema

| Keep Field | SkyWalking Source | Notes |
|------------|-------------------|-------|
| `id` | `id` (alarm ID) | Prefixed with `skywalking-` |
| `name` | `scope` + `name` + `ruleName` | e.g. `[Service] payment-service: service_resp_time_rule` |
| `description` | `alarmMessage` | Full alarm message |
| `severity` | `tags.level` | CRITICAL/HIGH/WARNING/LOW/INFO; defaults to HIGH |
| `status` | Always `FIRING` | SkyWalking alarms are always active at delivery time |
| `lastReceived` | `startTime` (ms epoch) | Converted to ISO-8601 |
| `source` | `["skywalking"]` | |

### Supported Alarm Scopes

| SkyWalking Scope | Description |
|-----------------|-------------|
| `SERVICE` | Service-level metrics (response time, success rate, throughput) |
| `SERVICE_INSTANCE` | Individual service instance metrics |
| `ENDPOINT` | Individual API endpoint metrics |
| `DATABASE_ACCESS` | Database client metrics |
| `SERVICE_RELATION` | Service-to-service call metrics |
