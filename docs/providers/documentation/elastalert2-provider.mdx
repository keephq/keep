## Overview

[ElastAlert2](https://github.com/jertel/elastalert2) is an open-source framework for alerting on anomalies, spikes, and patterns of interest from data stored in Elasticsearch. It periodically queries Elasticsearch and sends alerts when conditions match your defined rules.

The Keep ElastAlert2 provider receives alerts via ElastAlert2's built-in **HTTP POST** alert type, which sends a JSON payload to a configured webhook URL whenever a rule triggers.

## Connecting ElastAlert2 to Keep

### Step 1: Configure the HTTP POST alerter

Add the HTTP POST configuration to your ElastAlert2 rule YAML file. You can set this globally in `config.yaml` or per-rule:

```yaml
# Rule configuration
name: my-alert-rule
type: frequency
index: nginx-logs-*

# ... your rule conditions ...

# Keep webhook configuration
alert:
  - post

http_post_url: "https://api.keephq.dev/alerts/event/elastalert2"
http_post_headers:
  X-API-KEY: "<your-keep-api-key>"
  Content-Type: application/json
http_post_all_values: true
```

### Step 2: Add severity to your rules (optional but recommended)

ElastAlert2 doesn't have a native severity field, but you can add one via a custom field:

```yaml
http_post_payload:
  severity: "critical"
  rule_name: "{{ rule_name }}"
```

Or use ElastAlert2's `priority` field if your Elasticsearch documents contain one.

### Step 3: Reload ElastAlert2

```bash
# If running via systemd
sudo systemctl reload elastalert2

# If running manually, restart the process
python -m elastalert.elastalert --config config.yaml --rule rules/my-rule.yaml
```

## Example Rule

```yaml
name: high-error-rate
type: frequency
index: nginx-logs-*
num_events: 100
timeframe:
  minutes: 5
filter:
  - term:
      status: 500

alert:
  - post

http_post_url: "https://api.keephq.dev/alerts/event/elastalert2"
http_post_headers:
  X-API-KEY: "<your-keep-api-key>"
http_post_all_values: true
http_post_payload:
  severity: "critical"
  alert_subject: "High HTTP 5xx error rate"
  alert_text: "More than 100 HTTP 500 errors detected in 5 minutes"
```

## Alert Payload

When a rule triggers, ElastAlert2 sends a JSON payload similar to:

```json
{
  "rule_name": "high-error-rate",
  "alert_subject": "High HTTP 5xx error rate",
  "alert_text": "More than 100 HTTP 500 errors detected in 5 minutes",
  "@timestamp": "2024-01-15T14:30:00.000Z",
  "severity": "critical",
  "_index": "nginx-logs-2024.01.15",
  "host": { "name": "web-server-01" },
  "num_hits": 127
}
```

## Severity Mapping

Add a `severity` field to your ElastAlert2 rule payloads. Keep maps it as follows:

| ElastAlert2 severity | Keep severity |
|----------------------|---------------|
| `critical`           | Critical      |
| `error` / `high`     | High          |
| `warning` / `warn`   | Warning       |
| `info` / `informational` | Info     |
| `low`                | Low           |
| _(unset or unknown)_ | Info          |
