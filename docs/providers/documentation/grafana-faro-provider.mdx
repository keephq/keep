## Overview

[Grafana Faro](https://grafana.com/oss/faro/) is an open-source web SDK for Real User Monitoring (RUM). It captures JavaScript exceptions, performance traces, and user session data from browser-based applications and forwards them to a backend collector (typically Grafana Alloy or the Grafana Cloud Frontend Observability service).

Keep integrates with the **Grafana Alerting** layer that sits on top of Faro data. Alert rules written against Loki (for Faro logs/exceptions) or Prometheus (for Faro metrics) are pulled into Keep and normalized into `AlertDto` objects.

## Authentication

| Field | Required | Description |
|-------|----------|-------------|
| `grafana_url` | Yes | Base URL of your Grafana instance (e.g. `https://your-org.grafana.net`) |
| `api_token` | Yes | Grafana service-account token with at least the **Viewer** role |
| `app_name` | No | Filter alerts to a specific Faro `app_name`. Leave empty to receive all alerts. |

## Receiving Webhooks

For real-time alerting you can configure Grafana Alerting to push notifications to Keep:

1. In Grafana, navigate to **Alerting â†’ Contact points** and add a new **Webhook** contact point.
2. Set the URL to:
   ```
   https://<your-keep-host>/api/v1/providers/<provider-id>/alerts
   ```
3. Save and attach the contact point to an alert rule that references your Faro data source.

Keep will parse the [Grafana Alertmanager webhook payload](https://grafana.com/docs/grafana/latest/alerting/manage-notifications/webhook-notifier/) and create or update alerts automatically.

## Limitations

- Keep queries the Grafana Alerting API, **not** the Faro collector directly. Alerts only appear in Keep when a Grafana alerting rule has fired against Faro-sourced data.
- The optional `app_name` filter matches on the `app_name` or `app` label attached to alert rules.
- For self-hosted Grafana with Faro, ensure the Grafana instance is reachable from the Keep server.
