## Connecting with the Provider

The SolarWinds provider allows you to receive alerts from SolarWinds Orion monitoring
platform within Keep. It supports both API polling via the SWIS REST API and real-time
webhook delivery via SolarWinds alert actions.

## Setup

### Prerequisites

1. Access to a SolarWinds Orion instance
2. A SolarWinds user account with permissions to query alerts
3. Network access to the SWIS REST API port (default: 17774)
4. Keep instance with webhook capability (for real-time alerts)

### Configuration

The provider requires the following configuration:

```yaml
authentication:
  host_url: "https://orion.example.com:17774" # SolarWinds Orion SWIS endpoint
  username: "your-username" # SolarWinds Orion username
  password: "your-password" # SolarWinds Orion password
  verify_ssl: true # Set to false for self-signed certs
```

**Note:** SolarWinds Orion commonly uses self-signed SSL certificates. If you
encounter SSL verification errors, set `verify_ssl` to `false`.

### How It Works (API Polling)

The provider queries active alerts from SolarWinds Orion using the SWIS REST API
with SWQL (SolarWinds Query Language). It joins three Orion entities:

- **Orion.AlertActive** — currently active alert instances
- **Orion.AlertObjects** — the entity (node, interface, volume) that triggered the alert
- **Orion.AlertConfigurations** — the alert definition (name, severity, description)

### Webhook Configuration

To receive alerts in real-time, configure an HTTP POST alert action in SolarWinds Orion:

1. In the Orion web console, go to **Alerts & Activity > Alerts > Manage Alerts**.
2. Edit the alert definition you want to forward to Keep.
3. Under **Trigger Actions**, click **Add Action** and select
   **Send a GET or POST Request to a URL**.
4. Configure the action:
   - **URL**: Your Keep webhook URL (e.g., `https://keep.example.com/api/alerts/event/solarwinds`)
   - **Method**: `POST`
   - **Content Type**: `application/json`
   - **Header Name**: `X-API-KEY`
   - **Header Value**: Your Keep API key
   - **Body**:

```json
{
  "alert_name": "${N=Alerting;M=AlertName}",
  "alert_message": "${N=Alerting;M=AlertMessage}",
  "severity": "${N=Alerting;M=Severity}",
  "alert_active_id": "${N=Alerting;M=AlertActiveID}",
  "alert_object_id": "${N=Alerting;M=AlertObjectID}",
  "alert_id": "${N=Alerting;M=AlertID}",
  "object_name": "${N=SwisEntity;M=Caption}",
  "object_type": "${N=Alerting;M=ObjectType}",
  "node_name": "${N=SwisEntity;M=Node.Caption}",
  "ip_address": "${N=SwisEntity;M=IP_Address}",
  "triggered_datetime": "${N=Alerting;M=AlertTriggerTime;F=DateTime}",
  "acknowledged": "${N=Alerting;M=Acknowledged}",
  "acknowledged_by": "${N=Alerting;M=AcknowledgedBy}",
  "notification_type": "PROBLEM"
}
```

5. (Recommended) Add a **Reset Action** with the same configuration but change
   `"notification_type"` to `"RECOVERY"` to auto-resolve alerts in Keep.

### Severity Mapping

SolarWinds alert severities are automatically mapped to Keep severities:

<TableWrapper>
#### Severity Mapping
| SolarWinds Severity | Keep Severity |
|:--------------------|:--------------|
| Critical (0)        | CRITICAL      |
| Serious (1)         | HIGH          |
| Warning (2)         | WARNING       |
| Informational (3)   | INFO          |
| Notice (4)          | LOW           |

</TableWrapper>
