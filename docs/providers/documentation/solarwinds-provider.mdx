## Overview

The SolarWinds provider integrates Keep with SolarWinds Orion. It can pull active alerts and node status from the Orion platform using the SolarWinds Information Service (SWIS) REST API, and it can also receive alert notifications via webhooks.

## Connecting SolarWinds to Keep

### Option 1: Pulling alerts from SolarWinds

1. In the Keep UI, go to **Providers** and add a new **SolarWinds** provider.

2. Fill in the required fields:
   - **Host URL**: Your SolarWinds Orion server URL (e.g. `https://orion.example.com:17778`)
   - **Username**: A SolarWinds Orion user with API access
   - **Password**: The password for that user

3. Optionally, uncheck **Verify SSL** if your server uses a self-signed certificate.

4. Click **Connect**. Keep will now poll SolarWinds for active alerts and down nodes.

### Option 2: Receiving alerts via webhook

1. In SolarWinds Orion, go to **Settings > All Settings > Alerts & Reports > Alerts**.

2. Edit (or create) an alert and go to the **Trigger Actions** or **Reset Actions** tab.

3. Add a **Send a GET or POST Request to a Web Server** action.

4. Set the URL to your Keep webhook endpoint:

```
https://api.keephq.dev/alerts/event/solarwinds
```

5. Choose **POST** as the method and set the body to include relevant alert variables, for example:

```json
{
  "AlertName": "${AlertName}",
  "AlertMessage": "${AlertMessage}",
  "NodeName": "${NodeName}",
  "IP_Address": "${IP_Address}",
  "Severity": "${Severity}",
  "AlertObjectID": "${AlertObjectID}",
  "AlertActiveID": "${AlertActiveID}",
  "TriggeredDateTime": "${AlertTriggerTime}",
  "NotificationType": "trigger"
}
```

6. For **Reset Actions**, use the same URL but set `"NotificationType": "reset"` so Keep knows the alert has been resolved.

7. Add the Keep API key as a header: `X-API-KEY: <your-keep-api-key>`.
