## Overview

[Cribl](https://cribl.io/) is an observability pipeline platform that routes, transforms, and enriches telemetry data (logs, metrics, traces) from any source to any destination. The Keep integration works in two modes:

- **Push (webhook) mode** — Cribl's *HTTP Destination* forwards processed events to Keep's webhook endpoint in real time. Each JSON object (or array of objects) sent by Cribl is converted to an alert in Keep.
- **Pull mode** — Keep queries the Cribl Stream REST API to retrieve active search-job results and system notifications.

## Connecting Cribl to Keep

### Prerequisites

You need:
- A running Cribl Stream, Edge, or Cloud instance.
- Either a Cribl **API token** (Settings → API Tokens) or admin **username/password** credentials for self-hosted instances.

### Push (Webhook) Mode

1. In Cribl, open **Routing** (Stream) or **Processing** (Edge) and go to **Destinations**.
2. Click **+ Add Destination** and select **HTTP**.
3. Set the **URL** to your Keep webhook endpoint (shown in the provider setup dialog).
4. Under **Authentication**, choose **Manual** and add a request header:
   - **Header name**: `x-api-key`
   - **Value**: your Keep API key
5. Set **Backpressure Behavior** to *Block* or *Drop* as appropriate.
6. **Save** the destination and attach it to the desired pipeline or route.

Keep will receive events in real time. Each JSON object sent by Cribl is converted to an alert.

> **Tip** — to filter only specific events, add a Cribl *Filter* expression on the route or output group (e.g. `_path == '/logs/alerts'`).

### Pull Mode

Pull mode is enabled automatically when you configure the provider with valid credentials and a reachable `deployment_url`. Keep will periodically query `/api/v1/jobs` for completed or running search jobs and retrieve their results.
