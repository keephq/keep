## Connecting with the Provider

To connect Keep to OpsLevel, you need an API token:

1. Log in to your [OpsLevel](https://app.opslevel.com) account.
2. Navigate to **Settings** → **API Tokens**.
3. Click **Create API Token**, give it a name (e.g., "Keep Integration"), and copy the token.
4. Enter the token as `api_token` when configuring the provider in Keep.

Keep uses the OpsLevel GraphQL API to fetch services and their check results. Services with failing checks will appear as active alerts.

## Webhook Integration

To receive real-time check result notifications in Keep:

1. In OpsLevel, navigate to **Settings** → **Integrations** → **Webhooks**.
2. Click **Add Webhook**.
3. Enter your Keep webhook URL: `https://<your-keep-url>/alerts/event/opslevel`.
4. Select the check result events to forward (e.g., **Check Failed**, **Check Passed**).
5. Click **Save**.

Keep will automatically create and resolve alerts as OpsLevel check results change.

## Severity Mapping

Service alerts are derived from the service's maturity level in OpsLevel:

| OpsLevel Maturity Level | Keep Severity |
|------------------------|---------------|
| Beginner (level 0) | Critical |
| Bronze (level 1) | High |
| Silver (level 2) | Warning |
| Gold+ (level 3+) | Info |

For webhook check results, severity is derived from check category:

| OpsLevel Check Category | Keep Severity |
|------------------------|---------------|
| Security | Critical |
| Reliability, Performance | High |
| Quality, Custom | Warning |
| General | Info |
