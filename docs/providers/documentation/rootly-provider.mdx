## Overview

[Rootly](https://rootly.com/) is an incident management platform used by teams to manage incidents,
on-call rotations, and post-mortems. This provider brings Rootly alerts and incidents into Keep
for unified monitoring alongside your other alerting sources.

## Features

- **Alert Pulling**: Fetches alerts from Rootly's Alerts API with pagination
- **Incident Pulling**: Optionally fetches incidents from Rootly's Incidents API
- **Webhook Support**: Handles real-time events for alerts and incidents (created, updated, resolved, cancelled)
- **Incident Creation**: Create new incidents via Keep workflows
- **Alert Resolution**: Resolve or acknowledge alerts programmatically
- **Noise Suppression**: Rootly's noise classification maps to Keep's Suppressed status

## Connecting with the Provider

1. Log in to your [Rootly Dashboard](https://rootly.com/).
2. Navigate to **Organization Settings** → **API Keys**.
3. Click **Generate New API Key**.
4. Copy the API key and use it in the provider configuration.

### Configuration

```yaml
authentication:
  api_key: "your-rootly-api-key"
  api_url: "https://api.rootly.com"  # optional, defaults to this
  pull_incidents: true                # optional, also pull incidents
```

## Webhook Setup

To receive real-time events from Rootly:

1. In your Rootly dashboard, go to **Settings** → **Integrations** → **Webhooks**.
2. Add a new webhook endpoint pointing to your Keep instance:
   ```
   https://your-keep-instance/alerts/event/rootly
   ```
3. Select the events you want to receive:
   - `alert.created`, `alert.updated`, `alert.resolved`
   - `incident.created`, `incident.updated`, `incident.mitigated`, `incident.resolved`, `incident.cancelled`

## Outputs

The provider returns alerts and incidents as Keep AlertDtos with rich metadata including:
- Fingerprint-based deduplication (`rootly-alert-{id}` / `rootly-incident-{id}`)
- Service and environment extraction from Rootly data
- Labels including source type, services, groups, and Slack channel info
- Markdown-formatted descriptions
