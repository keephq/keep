## Overview

Zulip Provider allows you to send alert messages from Keep via [Zulip](https://zulip.com/), an open-source team chat application that combines the immediacy of real-time chat with the benefits of threaded conversations. The Zulip provider supports sending messages to both streams (channels) and individual users.

## Connecting with Zulip Provider

1. Create a Zulip bot from your Zulip server's **Settings → Bots → Add a new bot**.
2. Copy the bot's **Email** and **API key**.
3. Note your Zulip server URL (e.g. `https://zulip.example.com`).
4. Add Zulip Provider in Keep with the email, API key, and Zulip URL.
5. Connect the Zulip Provider with Keep.

## Message Format Support

### Stream Messages
Send messages to a Zulip stream with a topic:

```yaml
with:
  message: "🚨 Alert: High CPU usage detected on server-01"
  to: "alerts"
  topic: "infrastructure"
  type: "stream"
```

### Private Messages
Send private messages to a specific user:

```yaml
with:
  message: "Critical alert: Database connection failed"
  to: "user@example.com"
  type: "private"
```

### Default Topic
If no topic is provided, messages default to the "alerts" topic:

```yaml
with:
  message: "Maintenance window starting in 10 minutes"
  to: "general"
```

## Supported Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `message` | string | Yes | The message content (supports Markdown) |
| `to` | string | Yes | Stream name or user email address |
| `topic` | string | No | Topic name for stream messages (default: "alerts") |
| `type` | string | No | Message type: "stream" or "private" (default: "stream") |

## Example Workflow

```yaml
id: zulip-example
name: Zulip Example
description: Send a test notification to Zulip
triggers:
  - type: manual

actions:
  - name: send-zulip-message
    provider:
      type: zulip
      config: "{{ secrets.ZULIP }}"
    with:
      message: "🚨 Alert: Database connection failed on {{ event.host }}"
      to: "engineering-alerts"
      topic: "production"
```
