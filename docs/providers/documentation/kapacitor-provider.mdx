## Overview

[Kapacitor](https://docs.influxdata.com/kapacitor/v1/) is InfluxData's real-time data processing engine. It reads data from InfluxDB and can trigger alerts based on customizable thresholds and conditions defined in TICKscripts.

The Keep Kapacitor provider receives alerts via Kapacitor's built-in **HTTP Post** event handler, which forwards alert payloads to a configured webhook URL whenever an alert fires, changes state, or recovers.

## Connecting Kapacitor to Keep

### Method 1: Inline `.post()` in a TICKscript

Add an `.httpPost()` call directly in your alert chain:

```javascript
stream
  |from()
    .measurement('cpu')
  |window()
    .period(10s)
    .every(10s)
  |mean('usage_idle')
  |alert()
    .crit(lambda: "mean" < 10)
    .warn(lambda: "mean" < 20)
    .message('{{ .Level }}: CPU usage is at {{ index .Fields "mean" | printf "%.2f" }}%')
    .httpPost('https://api.keephq.dev/alerts/event/kapacitor')
```

### Method 2: Named HTTP Post handler

Define a reusable handler via the Kapacitor API or configuration file, then reference it in your TICKscripts.

**kapacitor.conf:**
```toml
[[httppost]]
  endpoint = "keep"
  url = "https://api.keephq.dev/alerts/event/kapacitor"
  [httppost.headers]
    "X-API-KEY" = "<your-keep-api-key>"
```

**TICKscript:**
```javascript
|alert()
  .id('{{ .Name }}/{{ .TaskName }}')
  .post('keep')
```

### Method 3: Via the Kapacitor API

```bash
kapacitor define-handler \
  -type post \
  -url https://api.keephq.dev/alerts/event/kapacitor \
  -header 'X-API-KEY=<your-keep-api-key>'
```

## Alert Payload

Kapacitor sends JSON payloads with the following structure:

```json
{
  "id": "cpu/cpu-total/alert",
  "message": "cpu alert triggered",
  "details": "CPU usage_idle dropped below threshold.",
  "time": "2024-01-15T10:30:00Z",
  "duration": 60000000000,
  "level": "CRITICAL",
  "name": "cpu",
  "taskName": "cpu_alert",
  "data": {
    "series": [
      {
        "name": "cpu",
        "tags": { "host": "web-01", "cpu": "cpu-total" },
        "columns": ["time", "usage_idle"],
        "values": [["2024-01-15T10:30:00Z", 2.5]]
      }
    ]
  }
}
```
