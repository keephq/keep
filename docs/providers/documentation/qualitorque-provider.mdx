## Overview

The QualiTorque (Torque) provider enables integration between Keep and QualiTorque, a cloud infrastructure automation platform. It supports both push (webhook) and pull (REST API) modes:

- **Webhook (push):** Receive real-time notifications for environment lifecycle events such as launches, deployments, errors, drift detection, and teardowns.
- **API (pull):** Periodically fetch environment statuses and surface environments in error or warning states.

## Connecting QualiTorque to Keep

### Option 1: Webhook (Push Mode)

Configure Torque to send notifications to Keep via webhook:

1. Log in to your [QualiTorque portal](https://portal.qtorque.io).
2. Navigate to **Administration > Notifications**.
3. Click **Add Notification Target** and choose **Webhook**.
4. Set the Webhook URL to your Keep webhook endpoint: `https://api.keephq.dev/alerts/event/qualitorque`.
5. Add a custom header: `X-API-KEY` with your Keep API key (generate one in [Keep settings](https://platform.keephq.dev/settings?selectedTab=users&userSubTab=api-keys)).
6. Select the event types you want to receive (e.g., Environment Launched, Environment Error, Drift Detected).
7. Save the notification target.

### Option 2: API (Pull Mode)

Connect Keep to Torque's REST API to poll environment statuses:

1. In Keep, go to **Providers** and add a new **QualiTorque** provider.
2. Enter your Torque API host (default: `https://portal.qtorque.io`).
3. Enter your Torque **Space** name.
4. Enter a long-lived **API Token** (generate one in Torque under **Administration > API Tokens**).
5. Save the provider configuration.

Keep will periodically check for environments in error, failed, or drift states and create alerts.

## Supported Events

| Event | Severity | Status |
|-------|----------|--------|
| Environment Error | Critical | Firing |
| Environment Failed | Critical | Firing |
| Force Ended | High | Firing |
| Ending Failed | High | Firing |
| Drift Detected | Warning | Firing |
| Active with Error | Warning | Firing |
| Launching / Deploying | Info | Acknowledged |
| Active | Info | Resolved |
| Ended | Info | Resolved |
