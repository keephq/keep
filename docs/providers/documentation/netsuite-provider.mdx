## Overview

NetSuite is Oracle's cloud-based ERP platform used by thousands of businesses for financials, inventory, CRM, and support. The Keep NetSuite provider integrates with the [NetSuite REST Record API](https://docs.oracle.com/en/cloud/saas/netsuite/ns-online-help/chapter_1540391670.html) using OAuth 1.0 Token-Based Authentication (TBA).

With this provider you can:
- Pull open support cases into Keep as alerts
- Create new support cases from Keep workflows

## Prerequisites

- NetSuite account with SuiteCloud access
- An **Integration Record** created in NetSuite (for OAuth consumer key/secret)
- An **Access Token** created via Setup > Users/Roles > Access Tokens

## Connecting NetSuite to Keep

### Step 1: Create an Integration Record

1. In NetSuite, go to **Setup > Integration > Manage Integrations > New**.
2. Give it a name (e.g. `Keep Integration`).
3. Enable **Token-Based Authentication**.
4. Save — copy the **Consumer Key** and **Consumer Secret** shown on the confirmation page (they are only shown once).

### Step 2: Create an Access Token

1. Go to **Setup > Users/Roles > Access Tokens > New**.
2. Select the Integration you created above.
3. Select the User (a dedicated service user is recommended).
4. Select a Role with access to Support Cases.
5. Save — copy the **Token ID** and **Token Secret** shown on the confirmation page.

### Step 3: Find your Account ID

Your Account ID is visible in the NetSuite URL. For example, if your URL is `https://1234567.app.netsuite.com`, your Account ID is `1234567`. For sandbox, it would be `1234567_SB1`.

### Step 4: Add the Provider in Keep

1. In Keep, go to **Providers** and add **NetSuite**.
2. Fill in the fields:
   - **Account ID**: e.g. `1234567`
   - **Consumer Key**: from Step 1
   - **Consumer Secret**: from Step 1
   - **Token ID**: from Step 2
   - **Token Secret**: from Step 2
3. Click **Connect**.
