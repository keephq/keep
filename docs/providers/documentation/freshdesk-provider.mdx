## Connecting with the Provider

To connect Keep to Freshdesk, you need your API key and your Freshdesk subdomain:

1. Log in to your [Freshdesk](https://freshdesk.com) account.
2. Click your profile avatar in the top-right corner and select **Profile Settings**.
3. Scroll down to find your **API Key** on the right side of the page.
4. Your Freshdesk domain is the subdomain from your Freshdesk URL: `https://<domain>.freshdesk.com`.
5. Enter the API key and domain when configuring the provider in Keep.

## Webhook Integration

To receive real-time ticket notifications in Keep:

1. In Freshdesk, navigate to **Admin** → **Automation** → **Webhooks** (or **Scenario Automations**).
2. Create a new automation rule that triggers on ticket events (created, updated, resolved, etc.).
3. Add a webhook action pointing to: `https://<your-keep-url>/alerts/event/freshdesk`.
4. Set the request body to send ticket details including `ticket_id`, `ticket_subject`, `ticket_status`, `ticket_priority`, `ticket_url`, and `ticket_description`.
5. Save the automation rule.

Keep will automatically create, update, and resolve alerts as Freshdesk tickets change.
