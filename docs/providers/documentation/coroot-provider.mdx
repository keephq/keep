## Connecting with the Provider

### Self-hosted Coroot

1. Navigate to your Coroot instance (e.g. `https://coroot.my-company.com`).
2. Open **Settings > API Keys** and generate a new key.
3. Set `coroot_url` to your instance's base URL and `api_key` to the key you just generated.
4. Set `project_id` to the Coroot project you want to monitor (visible in the URL as `/p/<project_id>/`).

### Coroot Cloud

Use your Coroot Cloud workspace URL and a Cloud API key with the same fields above.

## What Keep Monitors

Keep polls two Coroot data sources:

- **SLO violations** — any SLO that is not meeting its objective surfaces as a `CRITICAL` or `WARNING` alert depending on the severity Coroot reports.
- **Application health issues** — applications flagged as `degraded` or `unhealthy` by Coroot's built-in health checks are surfaced with their individual issues.

Healthy SLOs and applications are silently ignored.
