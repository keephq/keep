## Overview

The PRTG Provider integrates Keep with [PRTG Network Monitor](https://www.paessler.com/prtg) in two modes:

- **Pull mode** — Keep polls the PRTG REST API (`/api/table.json`) for sensors in a problem state (Down, Warning, Unusual, Partial Down) on a schedule.
- **Push mode (webhook)** — PRTG sends HTTP notification payloads to Keep's inbound webhook endpoint when a sensor changes state.

## Authentication

PRTG uses username + passhash (recommended) or username + password authentication.

| Field | Required | Description |
|-------|----------|-------------|
| `url` | Yes | PRTG server URL, e.g. `https://prtg.example.com` |
| `username` | Yes | PRTG username |
| `passhash` | No* | Passhash from PRTG → Account Settings → My Account → Passhash |
| `password` | No* | PRTG password (used only when `passhash` is not set) |
| `verify_ssl` | No | Verify SSL certificates (default: `true`). Set to `false` for self-signed certs. |

\* Either `passhash` or `password` must be provided.

**To find your passhash:** Log into PRTG → click your username → Account Settings → scroll to **Passhash**. Using the passhash is more secure than embedding your password.

## Push Mode: Connecting PRTG via Webhook

1. In PRTG, go to **Setup** → **Account Settings** → **Notifications**.
2. Click **Add new notification** and give it a name (e.g. "Send to Keep").
3. Enable **Execute HTTP Action**.
4. Set **URL** to `{keep_webhook_api_url}`.
5. Set **HTTP Method** to `POST` and **Content Type** to `application/json`.
6. In the **Payload** field, paste the following template:

```json
{
  "sensor": "%sensor",
  "device": "%device",
  "group": "%group",
  "probe": "%probe",
  "status": "%status",
  "message": "%message",
  "datetime": "%datetime",
  "sensor_id": "%sensorid",
  "device_id": "%deviceid",
  "group_id": "%groupid",
  "priority": "%priority",
  "lastvalue": "%lastvalue"
}
```

7. Click **Save**. Add this notification to sensor **Notification Triggers** as needed.

## Alert Mapping

### Status Mapping

| PRTG Status | Keep Status |
|-------------|-------------|
| Up | Resolved |
| Down | Firing |
| Down (Acknowledged) | Acknowledged |
| Warning | Firing |
| Unusual | Firing |
| Partial Down | Firing |
| Paused / Paused by User / Schedule / Dependency | Suppressed |
| Unknown | Pending |

### Severity Mapping

| PRTG Status | Keep Severity |
|-------------|---------------|
| Down / Down (Acknowledged) | Critical |
| Partial Down | High |
| Warning / Unusual | Warning |
| Paused / Up / Unknown | Info |

In push mode, the numeric PRTG priority (`%priority`) can further escalate severity (but never downgrade it):

| PRTG Priority | Keep Severity |
|---------------|---------------|
| 5 (`*****`) | Critical |
| 4 (`****`) | High |
| 3 (`***`) | Warning |
| 2 (`**`) | Info |
| 1 (`*`) | Low |
