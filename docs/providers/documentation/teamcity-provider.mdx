## Connecting with the Provider

To connect Keep to TeamCity, you need a TeamCity Access Token and your server URL:

1. Log in to your TeamCity server.
2. Click on your profile icon (top-right) and select **Profile**.
3. Navigate to the **Access Tokens** tab.
4. Click **Create access token**, give it a name, and click **Generate**.
5. Copy the generated token — store it securely.
6. Use your TeamCity server URL (e.g. `https://teamcity.example.com`) as the `host` and the token as the `token` when configuring the provider in Keep.

## Webhook Integration

To receive real-time build notifications in Keep:

1. In TeamCity, go to **Administration** → **Server Administration** → **Integrations**.
2. Select **Webhooks** or use a **Build Feature** on your build configuration.
3. Add a new webhook with the URL: `https://<your-keep-url>/alerts/event/teamcity`.
4. Select the events to forward: **Build Failed**, **Build Started**, **Build Successful**.
5. Save the webhook configuration.

Keep will parse the incoming TeamCity build payloads and create alerts for failed builds automatically.
