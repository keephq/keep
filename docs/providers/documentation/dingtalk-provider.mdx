## Overview

DingTalk Provider allows you to send alert messages from Keep via [DingTalk](https://www.dingtalk.com/) (钉钉), an enterprise messaging and collaboration platform widely used in China. DingTalk robot webhooks enable you to send text or markdown messages directly to group chats.

The DingTalk provider supports text and markdown messages with optional @mentions, making it a versatile choice for alert notifications in teams that use DingTalk for communication.

## Connecting with DingTalk Provider

1. Open a group chat in DingTalk and go to **Group Settings → Smart Assistant → Add Robot → Custom Robot**.
2. Copy the webhook URL which contains the `access_token`.
3. Optionally enable signature verification and copy the `secret`.
4. Add DingTalk Provider in Keep with the access token and optional secret.
5. Connect the DingTalk Provider with Keep.

## Message Format Support

### Text Messages
Send plain text messages:

```yaml
with:
  message: "Alert: High CPU usage detected on server-01"
```

### Markdown Messages
Send formatted markdown messages with a title:

```yaml
with:
  msgtype: "markdown"
  title: "Critical Alert"
  message: "### 🚨 Critical Alert\n\nDatabase connection failed on {{ event.host }}"
```

### Messages with @Mentions
Mention specific users by mobile number or user ID:

```yaml
with:
  message: "Alert: Server overload detected @18200000000"
  at_mobiles:
    - "18200000000"
```

### @All Members
Notify all members in the group:

```yaml
with:
  message: "Emergency: Production incident detected"
  is_at_all: true
```

## Supported Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `message` | string | Yes | The message content |
| `title` | string | No | Title for markdown messages |
| `msgtype` | string | No | Message type: `text` (default) or `markdown` |
| `at_mobiles` | list | No | List of mobile numbers to @mention |
| `at_userids` | list | No | List of user IDs to @mention |
| `is_at_all` | boolean | No | Whether to @all members |

## Example Workflow

```yaml
id: dingtalk-example
name: DingTalk Example
description: Send a test notification to DingTalk
triggers:
  - type: manual

actions:
  - name: send-dingtalk-message
    provider:
      type: dingtalk
      config: "{{ secrets.DINGTALK }}"
    with:
      message: "🚨 Alert: Database connection failed on {{ event.host }}"
      msgtype: "markdown"
      title: "Database Alert"
```
