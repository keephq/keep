## Authentication Parameters

The GotifyProviderAuthConfig class takes the following parameters:

- **url** (str): The URL of your Gotify server instance (e.g., `https://gotify.example.com` or `http://localhost:8080`). **Required**
- **token** (str): The Gotify application token used to authenticate API requests. **Required**

## Connecting with the Provider

1. Install Gotify or use an existing server. See [Gotify Server Setup](https://gotify.net/docs/install).
2. Create an application in the Gotify web UI and copy the generated **Application Token**.
3. Add the Gotify Provider in Keep with the server URL and token.

## Example

```yaml
id: gotify-example
name: Gotify Example
description: Send a test notification to Gotify
triggers:
  - type: manual

actions:
  - name: send-gotify-message
    provider:
      type: gotify
      config: "{{ secrets.GOTIFY }}"
    with:
      title: "🚨 Keep Alert"
      message: "High CPU usage detected on {{ event.host }}"
      priority: 8
```

## Supported Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `message` | string | Yes | The message body to send |
| `title` | string | No | The notification title |
| `priority` | int | No | Message priority: 0=low, 5=normal (default), 10=high |
