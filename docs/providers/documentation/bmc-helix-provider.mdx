## Overview

The BMC Helix ITSM Provider integrates Keep with [BMC Helix ITSM](https://www.bmc.com/it-solutions/bmc-helix-itsm.html) (formerly Remedy), one of the most widely deployed enterprise ITSM platforms.

Four capabilities are supported:

- **Pull mode** — Keep polls open/active incidents from the Helix ITSM REST API on a schedule.
- **Push mode (webhook)** — Helix sends HTTP notifications to Keep's inbound webhook endpoint when incidents are created or updated.
- **Incident creation** — Keep workflows can create new incidents in Helix ITSM via the `notify()` action.
- **Service topology** — Keep can pull CMDB CI relationships from the Asset Management module to build a service dependency map.

## Authentication

BMC Helix ITSM uses JWT bearer token authentication (Innovation Suite ≥ 18.x, Helix Cloud).

| Field | Required | Description |
|-------|----------|-------------|
| `base_url` | Yes | Helix ITSM base URL, e.g. `https://your-company.onbmc.com` |
| `username` | Yes | BMC Helix username |
| `password` | Yes | BMC Helix password |
| `verify_ssl` | No | Verify SSL certificate (default: `true`). Set to `false` for self-signed certs. |
| `timeout` | No | HTTP request timeout in seconds (default: `30`). |

The provider automatically exchanges credentials for a JWT token via `POST /api/jwt/login`. The token is cached for the session lifetime.

## Pull Mode

Keep calls `GET /api/arsys/v1/entry/HPD:IncidentInterface` with a qualification filter that excludes `Closed` and `Cancelled` incidents. The following fields are returned per incident:

- Incident Number, Summary, Notes, Status, Priority
- Urgency, Impact, Assigned Group, Assignee
- Company, Service, Category Tier 1
- Submit Date, Last Modified Date

## Push Mode (Webhook)

BMC Helix ITSM can be configured to send HTTP notifications to Keep when incidents change state.

### Option 1: HTTP-based Notification via Helix ITSM Email / HTTP Action

1. In Helix ITSM, go to **Application Administration** → **Filters** → add a filter on `HPD:IncidentInterface`.
2. Set the action to **Call Guide** or **HTTP Action** and point it to:
   ```
   {keep_webhook_api_url}
   ```
3. Pass the incident fields as a JSON body.

### Option 2: Impact Event Adapter

Configure a BMC Impact Event Adapter or Smart-IT notification rule to POST incident JSON to the Keep webhook endpoint when events match your criteria.

### Expected Payload

Keep supports two payload shapes:

**Wrapped** (mirrors the REST API response):
```json
{
  "values": {
    "Incident Number": "INC0001234",
    "Summary": "Disk full on DB server",
    "Status": "New",
    "Priority": "2",
    "Company": "Acme Corp",
    "Service": "Database Service",
    "Submit Date": "1700000000000"
  }
}
```

**Flat** (direct field map):
```json
{
  "Incident Number": "INC0001234",
  "Summary": "Disk full on DB server",
  "Status": "New",
  "Priority": "2"
}
```

## Service Topology (CMDB)

When topology pull is enabled, Keep queries `AST:AssetRelationship` for active CI relationships and builds a service dependency map displayed in the Keep Topology view.
