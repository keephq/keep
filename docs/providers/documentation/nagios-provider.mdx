## Connecting with the Provider

The Nagios provider supports Nagios XI REST API polling.

1. In Nagios XI, create or copy an API key for the user Keep should poll as.
2. Configure the provider with your Nagios XI URL and API key.
3. Keep polls host status and service status from the Nagios XI REST API.

The `host_url` can be either the server root, such as `https://nagios.example.com`, or the Nagios XI base path, such as `https://nagios.example.com/nagiosxi`.

## State Mapping

| Nagios state | Keep status | Keep severity |
| --- | --- | --- |
| 0 OK / UP | resolved | low |
| 1 WARNING | firing | warning |
| 2 CRITICAL / DOWN | firing | critical |
| 3 UNKNOWN | firing | info |

Acknowledged Nagios problems are ingested with Keep's acknowledged status.

## Troubleshooting

- Confirm the API key can read `/nagiosxi/api/v1/objects/hoststatus` and `/nagiosxi/api/v1/objects/servicestatus`.
- If provider validation fails, verify that the Nagios XI URL is reachable from the Keep backend.
- Nagios Core CGI polling and webhook notification commands are outside this provider's current scope.
