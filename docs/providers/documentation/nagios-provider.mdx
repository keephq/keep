## Overview

The Nagios provider enables integration between Keep and [Nagios](https://www.nagios.org/), a widely-used infrastructure monitoring tool. Alerts are received via webhook — Nagios sends HTTP POST notifications to Keep when host or service state changes occur.

## Connecting Nagios to Keep

### Step 1: Define a notification command

Add the following command definitions to your Nagios configuration (typically in `commands.cfg`):

**Service notification command:**

```cfg
define command {
    command_name    notify-keep-service
    command_line    /usr/bin/curl -s -X POST \
        -H "Content-Type: application/json" \
        -H "X-API-KEY: YOUR_KEEP_API_KEY" \
        -d '{ \
            "host_name": "$HOSTNAME$", \
            "host_address": "$HOSTADDRESS$", \
            "service_description": "$SERVICEDESC$", \
            "state": "$SERVICESTATE$", \
            "output": "$SERVICEOUTPUT$", \
            "notification_type": "$NOTIFICATIONTYPE$", \
            "timestamp": "$LONGDATETIME$", \
            "problem_acknowledged": "$SERVICEACKAUTHOR$" \
        }' \
        "YOUR_KEEP_WEBHOOK_URL"
}
```

**Host notification command:**

```cfg
define command {
    command_name    notify-keep-host
    command_line    /usr/bin/curl -s -X POST \
        -H "Content-Type: application/json" \
        -H "X-API-KEY: YOUR_KEEP_API_KEY" \
        -d '{ \
            "host_name": "$HOSTNAME$", \
            "host_address": "$HOSTADDRESS$", \
            "state": "$HOSTSTATE$", \
            "output": "$HOSTOUTPUT$", \
            "notification_type": "$NOTIFICATIONTYPE$", \
            "timestamp": "$LONGDATETIME$" \
        }' \
        "YOUR_KEEP_WEBHOOK_URL"
}
```

### Step 2: Create a contact for Keep

```cfg
define contact {
    contact_name                    keep
    alias                           Keep Webhook
    service_notification_commands   notify-keep-service
    host_notification_commands      notify-keep-host
    service_notification_period     24x7
    host_notification_period        24x7
    service_notification_options    w,u,c,r
    host_notification_options       d,u,r
}
```

### Step 3: Add the contact to a contact group

```cfg
define contactgroup {
    contactgroup_name   keep-notifications
    alias               Keep Notifications
    members             keep
}
```

### Step 4: Assign the contact group to hosts/services

Add `contact_groups keep-notifications` to the hosts or services you want to monitor through Keep.

### Step 5: Restart Nagios

```bash
sudo systemctl restart nagios
```

## Supported alert fields

| Field | Description |
|-------|-------------|
| `host_name` | Name of the host |
| `host_address` | IP address of the host |
| `service_description` | Name of the service (empty for host alerts) |
| `state` | Current state (OK, WARNING, CRITICAL, UNKNOWN for services; UP, DOWN, UNREACHABLE for hosts) |
| `output` | Plugin output text |
| `notification_type` | Type of notification (PROBLEM, RECOVERY, ACKNOWLEDGEMENT, etc.) |
| `timestamp` | Time of the event |
