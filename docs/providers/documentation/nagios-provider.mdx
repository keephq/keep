## Overview

Keep integrates with both **Nagios Core** (open-source) and **Nagios XI** (commercial) to centralise your infrastructure monitoring alerts.

| Mode | Nagios Core | Nagios XI |
|------|-------------|-----------|
| Push (webhook) | ✅ | ✅ |
| Pull (REST API) | ❌ | ✅ |

## Pull-Based Integration (Nagios XI)

To enable pull-based alert fetching, provide your Nagios XI base URL and API key in the provider configuration.

### Finding Your API Key

1. Log in to your Nagios XI web interface.
2. Navigate to **Admin > Manage API Keys**.
3. Create a new key or copy an existing one.

### What Gets Pulled

- **Host alerts**: All hosts in DOWN or UNREACHABLE state.
- **Service alerts**: All services in WARNING, CRITICAL, or UNKNOWN state.
- Acknowledged problems are surfaced with `acknowledged: true` so you can filter them in Keep.

## Push-Based Integration (Webhook)

Both Nagios Core and Nagios XI can push alerts to Keep in real time using curl-based notification commands.

### Step 1: Add Notification Commands

Add the following to your Nagios configuration (e.g. `/usr/local/nagios/etc/objects/commands.cfg`):

```cfg
define command {
    command_name    notify-keep-host
    command_line    /usr/bin/curl -s -X POST \
      -H "Content-Type: application/json" \
      -d '{"host_name": "$HOSTNAME$", "host_alias": "$HOSTALIAS$", \
"host_address": "$HOSTADDRESS$", "state": "$HOSTSTATE$", \
"state_type": "$HOSTSTATETYPE$", "output": "$HOSTOUTPUT$", \
"long_output": "$LONGHOSTOUTPUT$", "notification_type": "$NOTIFICATIONTYPE$", \
"timestamp": "$TIMET$"}' \
      "YOUR_KEEP_WEBHOOK_URL?api_key=YOUR_API_KEY"
}

define command {
    command_name    notify-keep-service
    command_line    /usr/bin/curl -s -X POST \
      -H "Content-Type: application/json" \
      -d '{"host_name": "$HOSTNAME$", "host_alias": "$HOSTALIAS$", \
"host_address": "$HOSTADDRESS$", "service_description": "$SERVICEDESC$", \
"state": "$SERVICESTATE$", "state_type": "$SERVICESTATETYPE$", \
"output": "$SERVICEOUTPUT$", "long_output": "$LONGSERVICEOUTPUT$", \
"notification_type": "$NOTIFICATIONTYPE$", "timestamp": "$TIMET$"}' \
      "YOUR_KEEP_WEBHOOK_URL?api_key=YOUR_API_KEY"
}
```

### Step 2: Create a Keep Contact

```cfg
define contact {
    contact_name                    keep
    alias                           Keep Alerting
    service_notification_period     24x7
    host_notification_period        24x7
    service_notification_options    w,u,c,r
    host_notification_options       d,u,r
    service_notification_commands   notify-keep-service
    host_notification_commands      notify-keep-host
}
```

### Step 3: Add the Contact to Your Hosts/Services

```cfg
define contactgroup {
    contactgroup_name   keep-notifications
    alias               Keep Notifications
    members             keep
}
```

Then add `contact_groups keep-notifications` to your host and service definitions.

### Step 4: Restart Nagios

```bash
sudo systemctl restart nagios
# or for Nagios XI:
sudo service nagios restart
```

## Alert Severity Mapping

| Nagios State | Keep Severity |
|---|---|
| OK / UP | INFO |
| WARNING | WARNING |
| CRITICAL / DOWN / UNREACHABLE | CRITICAL |
| UNKNOWN | INFO |

| Nagios State | Keep Status |
|---|---|

| WARNING / CRITICAL / UNKNOWN / DOWN / UNREACHABLE | Firing |
