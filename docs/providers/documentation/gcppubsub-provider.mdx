## Overview

Google Cloud Pub/Sub is a fully managed message-passing service. Many GCP services publish events to Pub/Sub topics, including:

- **GKE cluster notifications** — security bulletins, end-of-support warnings, auto-upgrade events
- **Cloud Monitoring** — alert notifications forwarded via a Notification Channel
- **Custom application events** — anything your services publish

The Keep GCP Pub/Sub provider polls a subscription, converts messages to alerts, and acknowledges them so they aren't redelivered.

## Prerequisites

- A GCP project with the Pub/Sub API enabled
- A **topic** that receives the events you care about
- A **subscription** attached to that topic (pull mode)
- A **Service Account** with `roles/pubsub.subscriber` on the subscription

## Connecting GCP Pub/Sub to Keep

### Step 1: Create a Pub/Sub Topic and Subscription

If you're capturing GKE notifications:

```bash
# Enable GKE notifications to a topic
gcloud container clusters update CLUSTER_NAME \
  --notification-config=pubsub=ENABLED,pubsub-topic=projects/PROJECT_ID/topics/TOPIC_NAME

# Create a pull subscription
gcloud pubsub subscriptions create keep-sub \
  --topic=TOPIC_NAME \
  --ack-deadline=60
```

For Cloud Monitoring, add a Pub/Sub Notification Channel pointing at your topic in the GCP Console under **Monitoring > Alerting > Notification Channels**.

### Step 2: Create a Service Account

```bash
gcloud iam service-accounts create keep-pubsub \
  --display-name="Keep Pub/Sub Reader"

gcloud pubsub subscriptions add-iam-policy-binding keep-sub \
  --member="serviceAccount:keep-pubsub@PROJECT_ID.iam.gserviceaccount.com" \
  --role="roles/pubsub.subscriber"
```

### Step 3: Generate a Service Account Key

```bash
gcloud iam service-accounts keys create key.json \
  --iam-account=keep-pubsub@PROJECT_ID.iam.gserviceaccount.com
```

### Step 4: Add the Provider in Keep

1. In Keep, go to **Providers** and add **GCP Pub/Sub**.
2. Upload or paste the contents of `key.json` as the **Service Account JSON**.
3. Fill in:
   - **Project ID**: your GCP project ID
   - **Subscription ID**: the subscription name (e.g. `keep-sub`)
   - **Max Messages**: how many messages to pull per cycle (default: 100)
4. Click **Connect**.

## Message Format

Messages are decoded from base64 and parsed as JSON when possible. The `notificationType` attribute (set by GKE and other GCP services) is used to classify severity:

| Notification Type | Keep Severity |
|---|---|
| `SecurityBulletinEvent` | Critical |
| `EndOfSupportEvent` | High |
| `UpgradeAvailableEvent` | High |
| Everything else | Info |
