<Tip>
  The AWS Bedrock Provider supports querying all major foundational models available through
  AWS Bedrock, including Anthropic Claude, Meta Llama, Amazon Titan, Mistral, and Cohere Command.
  It works with both IAM roles and explicit credentials.
</Tip>

## Overview

AWS Bedrock is a fully managed service that makes foundational models from leading AI companies
available through a single API. This provider integrates Bedrock with Keep, allowing you to
use AI models in your alerting and incident management workflows — for example, to summarize
alerts, classify incidents, or generate runbook steps.

## Connecting with the Provider

### Option 1: IAM Role (Recommended for AWS-hosted Keep)

If Keep is running on AWS (EC2, ECS, EKS, Lambda), you can use an IAM role attached to the
compute instance. No explicit credentials are needed — just set the `region`.

1. Create an IAM policy with Bedrock permissions:
```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "bedrock:InvokeModel"
      ],
      "Resource": "arn:aws:bedrock:*::foundation-model/*"
    }
  ]
}
```

2. Attach the policy to your EC2 instance profile, ECS task role, or EKS service account.

3. Configure the provider with only `region`:
```yaml
authentication:
  region: "us-east-1"
```

### Option 2: Explicit Credentials

For non-AWS environments, provide AWS access keys:

1. Create an IAM user with the Bedrock policy above.
2. Generate access keys:
```bash
aws iam create-access-key --user-name bedrock-user
```
3. Configure the provider:
```yaml
authentication:
  region: "us-east-1"
  access_key: "AKIAXXXXXXXXXXXXXXXX"
  secret_access_key: "xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
```

### Option 3: Temporary Credentials

For environments using AWS STS:
```yaml
authentication:
  region: "us-east-1"
  access_key: "ASIAXXXXXXXXXXXXXXXX"
  secret_access_key: "xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
  session_token: "FwoGZXIvYXdzE..."
```

## Outputs

The provider returns the model's text response based on the prompt provided. For Claude models,
structured JSON output is supported via the `structured_output_format` parameter.
