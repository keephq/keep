## Authentication Parameters

The WebexProviderAuthConfig class takes the following parameters:

- **access_token** (str): The Cisco Webex Bot Access Token used to authenticate API requests. **Required**

## Connecting with the Provider

1. Go to [Webex for Developers](https://developer.webex.com/) and create a bot or get an existing bot token.
2. Copy the **Bot Access Token**.
3. Add the Webex Provider in Keep with the access token.
4. Create a Webex space (room) and add your bot to it, or use `toPersonEmail` / `toPersonId` to send direct messages.

## Example

```yaml
id: webex-example
name: Webex Example
description: Send a test notification to Cisco Webex
triggers:
  - type: manual

actions:
  - name: send-webex-message
    provider:
      type: webex
      config: "{{ secrets.WEBEX }}"
    with:
      message: "🚨 Keep Alert: High CPU usage detected on {{ event.host }}"
      roomId: "{{ secrets.WEBEX_ROOM_ID }}"
      markdown: "**Alert**: High CPU usage detected on `{{ event.host }}`"
```

## Supported Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `message` | string | Yes | The message text to send |
| `roomId` | string | No* | The Webex room/space ID to send the message to |
| `toPersonEmail` | string | No* | The email address of the recipient |
| `toPersonId` | string | No* | The Webex person ID of the recipient |
| `markdown` | string | No | Markdown-formatted message (overrides `text` if provided) |

*At least one of `roomId`, `toPersonEmail`, or `toPersonId` is required.
