## Overview

[Grafana Tempo](https://grafana.com/oss/tempo/) is a high-scale distributed tracing backend compatible with Jaeger, Zipkin, and OpenTelemetry. It uses **TraceQL** — a purpose-built query language — to search across millions of traces.

The Keep Grafana Tempo provider queries your Tempo instance for:
- **Error traces** — root spans with `status=error`
- **Slow traces** — root spans whose duration exceeds a configurable threshold

These are surfaced as Keep alerts, allowing you to correlate tracing data with logs and metrics from other providers.

## Prerequisites

- A running Grafana Tempo instance (self-hosted or Grafana Cloud)
- Network access from Keep to the Tempo API endpoint (default port 3200)
- An API token or basic auth credentials (if Tempo is behind authentication)

## Connecting Grafana Tempo to Keep

### Self-Hosted Tempo

If Tempo is accessible without authentication (e.g., within your cluster):

1. In Keep, go to **Providers** and add **Grafana Tempo**.
2. Set **Host** to your Tempo URL, e.g. `http://tempo.monitoring.svc.cluster.local:3200`.
3. Leave **Token** and **Username** blank.
4. Adjust **Latency Threshold (ms)** and **Error Rate Threshold (%)** as needed.
5. Click **Connect**.

### Grafana Cloud Tempo

For Grafana Cloud, use HTTP Basic Auth:

1. In your Grafana Cloud stack, go to **Details** and find your **Tempo data source** — note the **Host** URL and the **Instance ID** (a number like `123456`).
2. Generate an API key in Grafana Cloud with the `MetricsPublisher` or `Viewer` role.
3. In Keep:
   - **Host**: your Grafana Cloud Tempo host (e.g. `https://tempo-prod-01-prod-us-east-0.grafana.net`)
   - **Username**: your numeric instance ID
   - **Token**: your Grafana Cloud API key
4. Click **Connect**.

### Tempo Behind a Grafana Instance

If you access Tempo through Grafana (via the datasource proxy), point Keep at the raw Tempo endpoint instead — the provider uses the Tempo API directly.

## Alert Behavior

| Condition | Keep Severity | Example |
|---|---|---|
| Root span with `status=error` | Critical | 500 from `/v1/payments/charge` |
| Duration > threshold (default 2000ms) | High | 4.8s database query in `inventory-service` |

Traces are identified by their **Trace ID** which is used as both the alert ID and fingerprint for deduplication.
