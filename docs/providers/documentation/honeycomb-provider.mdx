## Overview

The Honeycomb Provider integrates Keep with [Honeycomb](https://www.honeycomb.io/), an observability platform built for high-cardinality event data and distributed tracing. This integration enables you to:

- **Pull triggered alerts** (triggers) from Honeycomb into Keep as alerts
- Receive **real-time notifications** when Honeycomb triggers fire or resolve via webhooks
- Correlate Honeycomb trigger events with alerts from other observability tools in Keep

## Connecting with the Provider

1. Log in to your [Honeycomb account](https://ui.honeycomb.io/).
2. Navigate to **Account Settings** → **API Keys**.
3. Create a new API key with at least **Read** access, or use an existing key.
4. Copy the API key.
5. In Keep, create a new Honeycomb provider:
   - Enter your **API Key**
   - Optionally enter a specific **Dataset** slug (leave blank to query all datasets)

## Webhook Setup

To receive real-time trigger notifications from Honeycomb in Keep:

1. In Honeycomb, navigate to the trigger you want to connect.
2. In the trigger's **Recipients** section, click **Add Recipient**.
3. Select **Webhook** and enter your Keep webhook URL: `{keep_webhook_api_url}`.
4. Add a custom header `x-api-key` with your Keep API key.
5. Save the trigger.

Honeycomb will now send a POST request to Keep whenever the trigger fires or resolves.
