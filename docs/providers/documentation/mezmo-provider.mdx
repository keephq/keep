## Overview

[Mezmo](https://www.mezmo.com/) (formerly LogDNA) is a log management and observability platform that enables teams to collect, monitor, and analyse log data at scale.

The Mezmo provider connects Keep to Mezmo using Mezmo's **Webhook** alert channel. When a log-based alert rule triggers or recovers, Mezmo sends a JSON payload to Keep's inbound webhook URL.

No API credentials are required — Keep acts as a pure webhook receiver.

## Connecting Mezmo to Keep

1. Log in to [Mezmo](https://app.mezmo.com/).
2. Navigate to **Settings** → **Alerts** → **Alert Channels**.
3. Click **Add Channel** and choose **Webhook**.
4. Give the channel a name (e.g. "Keep").
5. Set the **URL** to the Keep webhook URL shown in your provider configuration.
6. Set **Method** to `POST` and **Content Type** to `application/json`.
7. Click **Save**.
8. Open any **Alert** rule and add the new webhook channel to the **Alert Channels** section.
9. Click **Save**.

Keep will now receive both firing and recovery notifications from Mezmo.

## Alert Mapping

### Status Mapping

| Condition | Keep Status |
|-----------|-------------|
| `resolved: true` | Resolved |
| `ended_at` is set | Resolved |
| All other active alerts | Firing |

### Severity Mapping

Mezmo severity strings and syslog numeric levels map to Keep severity:

| Mezmo Severity | Keep Severity |
|----------------|---------------|
| CRITICAL / fatal | Critical |
| ERROR / `3` | High |
| WARN / WARNING / `4` | Warning |
| INFO / `5` / `6` | Info |
| DEBUG / trace / `7` | Low |
| Emergency (`0`) / Alert (`1`) / Critical (`2`) | Critical |

Recovery alerts (resolved=true) always have **Info** severity in Keep, regardless of the original priority.
