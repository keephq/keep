## Overview

The Mezmo provider enables Keep to receive alert notifications from [Mezmo](https://www.mezmo.com) (formerly LogDNA). Mezmo fires webhook alerts when log search queries match at a configurable threshold — for example, "more than 50 ERROR-level log lines in 5 minutes."

This is a zero-configuration, webhook-only integration — no credentials or API polling required.

## How it works

```
Mezmo log ingestion
        ↓  alert threshold exceeded
  HTTP POST JSON payload
        ↓
  Keep webhook endpoint
```

## Expected JSON payload

Keep parses the following fields from the Mezmo webhook body:

| Field | Type | Description |
|-------|------|-------------|
| `alert_id` | string | Unique alert identifier |
| `name` / `title` | string | Name of the Mezmo view or alert rule |
| `description` | string | Summary of why the alert fired |
| `level` | string | Log level that triggered: `critical`, `error`, `warning`, `info`, `debug` |
| `timestamp` | string \| number | When the alert fired — ISO-8601 string or epoch milliseconds |
| `query` | string | The log search query |
| `url` | string | Deep-link to the Mezmo view |
| `app` | string | Application name from the matching logs |
| `host` | string | Host that generated the matching logs |
| `account` | string | Mezmo organization/account |
| `lines` | array | Matched log lines, each with `timestamp`, `app`, `level`, `line` |

Example payload:

```json
{
  "alert_id": "mezmo-alert-001",
  "name": "High error rate on payment-service",
  "description": "More than 50 ERROR logs matched in the last 5 minutes.",
  "level": "error",
  "timestamp": "2024-01-15T14:30:00Z",
  "query": "level:error app:payment-service",
  "url": "https://app.mezmo.com/org/views/abc123",
  "app": "payment-service",
  "host": "prod-payments-01",
  "account": "acme-corp",
  "lines": [
    {
      "timestamp": "2024-01-15T14:29:55Z",
      "app": "payment-service",
      "level": "error",
      "line": "NullPointerException in PaymentProcessor.charge() line 87"
    }
  ]
}
```

## Severity mapping

| Mezmo log level | Keep severity |
|----------------|--------------|
| `critical` | CRITICAL |
| `error` | HIGH |
| `warning` / `warn` | WARNING |
| `info` | INFO |
| `debug` / `trace` | LOW |

## Connecting Mezmo to Keep

### 1. Add the Mezmo provider in Keep

In the Keep UI, go to **Providers → Add Provider → Mezmo**. No credentials are needed — save it to get your webhook URL:

```
https://api.keephq.dev/alerts/event/mezmo
```

### 2. Create a webhook destination in Mezmo

1. Log into [Mezmo](https://app.mezmo.com) and open your **Organization Settings**.
2. Navigate to **Alerts**.
3. Click **Add Webhook** and enter:
   - **URL:** your Keep webhook URL from step 1
   - **Method:** POST
   - **Headers:** add `X-API-KEY` with your Keep API key as the value
4. Click **Save**.

### 3. Attach the webhook to a View alert

1. In Mezmo, go to the **Views** screen and open the view you want to alert on.
2. Click **Alerts** on the view and add or edit an alert.
3. Select your Keep webhook as the notification channel.
4. Configure the threshold (e.g., "50 lines in 5 minutes") and click **Save**.

### 4. Test the integration

Use **Send Test Alert** in Mezmo's webhook settings to send a sample payload. The alert should appear in Keep within seconds.
