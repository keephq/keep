## Overview

The Flowtriq Provider enables integration between Keep and [Flowtriq](https://flowtriq.com), a DDoS detection and network attack monitoring platform. Flowtriq analyzes network traffic flows to detect volumetric attacks, protocol-based attacks, and application-layer attacks in real time.

This provider supports:

- **Webhook ingestion**: Receive DDoS attack alerts from Flowtriq as they happen, mapped to Keep's alert schema with severity, status, and fingerprinting.
- **API polling**: Pull active incidents from the Flowtriq API.
- **Alert enrichment**: Query the Flowtriq API for detailed incident metadata (bandwidth, packet rates, attack vectors, source IP counts) to enrich alerts within Keep workflows.

## Connecting with the Provider

### API Key

1. Log in to your Flowtriq dashboard.
2. Navigate to **Settings > API Keys**.
3. Generate a new API key with read permissions.
4. Use this key when configuring the provider in Keep.

### Webhook Setup

To receive real-time DDoS alerts from Flowtriq:

1. In your Flowtriq dashboard, go to **Settings > Integrations > Webhooks**.
2. Add a new webhook pointing to your Keep webhook URL.
3. Include the `X-API-KEY` header with your Keep API key.
4. Select which alert events to forward (attack start, attack end, severity change).

## Alert Fields

Flowtriq webhook payloads use a nested structure with `incident` and `node` objects. The following fields are mapped to Keep's alert model:

| Flowtriq Field | Keep Field | Description |
|---|---|---|
| `incident.id` | `id` | Unique incident identifier (UUID) |
| `incident.title` | `name` | Human-readable alert name (falls back to attack_family + node IP) |
| `incident.severity` | `severity` | Incident severity (critical, high, warning, low, info) |
| `incident.status` | `status` | Incident status mapped to firing/resolved |
| `incident.description` | `description` | Incident summary with bandwidth and packet rate metrics |
| `incident.attack_family` | `labels.attack_family` | Attack classification (udp_flood, syn_flood, etc.) |
| `incident.peak_bps` | `peak_bps` | Peak attack bandwidth in bits per second |
| `incident.peak_pps` | `peak_pps` | Peak attack packet rate in packets per second |
| `incident.dashboard_url` | `url` | Link to the incident in the Flowtriq dashboard |
| `node.ip_address` | `labels.node_ip_address` | IP address of the target node |
| `node.name` | `labels.node_name` | Name of the target node |
