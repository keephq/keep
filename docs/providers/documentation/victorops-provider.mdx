## Overview

[Splunk On-Call](https://www.splunk.com/en_us/products/on-call.html) (formerly VictorOps) is an incident management platform that aggregates alerts from monitoring tools, routes them to the right on-call engineers, and orchestrates escalation policies.

Keep integrates with Splunk On-Call in two ways:

1. **Polling** — Keep queries the REST API for all active (TRIGGERED and ACKNOWLEDGED) incidents.
2. **Webhooks** — Keep accepts outbound webhook payloads from Splunk On-Call so that incident state changes appear in Keep in real time.

## Authentication

| Field | Required | Description |
|-------|----------|-------------|
| `api_id` | Yes | Splunk On-Call API ID |
| `api_key` | Yes | Splunk On-Call API Key |
| `organization_slug` | Yes | Your org slug (visible in the Splunk On-Call URL) |

### Finding your API credentials

1. Log in to the [Splunk On-Call portal](https://portal.victorops.com/).
2. Navigate to **Integrations → API**.
3. Note your **API ID** and generate (or copy) an **API Key**.
4. Your organization slug appears in the URL: `app.victorops.com/client/<your-slug>`.

## Receiving Webhooks

To receive real-time incident notifications:

1. In the Splunk On-Call portal, go to **Integrations → Outgoing Webhooks**.
2. Click **Add Webhook** and set the endpoint to:
   ```
   https://<your-keep-host>/api/v1/providers/<provider-id>/alerts
   ```
3. Select the events to forward (e.g. **Incident Triggered**, **Incident Acknowledged**, **Incident Resolved**).
4. Save the webhook.

Keep will parse the payload and create or update the corresponding alert automatically.

## Limitations

- The REST API requires an active Splunk On-Call subscription with API access enabled.
- Only incidents visible to the API credentials (API ID + Key) are returned.
- Historical (resolved) incidents older than the Splunk On-Call retention window are not available.
