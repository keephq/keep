## Overview

[Lark](https://www.larksuite.com/) (known as **Feishu** in mainland China) is an enterprise collaboration suite by ByteDance that combines messaging, video calls, cloud documents, and a built-in **Service Desk** (helpdesk) module.

The Keep Lark provider supports two integration modes:

| Mode | What it does |
|------|--------------|
| **Pull** | Periodically fetches open service desk tickets and surfaces them as alerts in Keep |
| **Webhook** | Receives real-time ticket lifecycle events (created, updated, closed) pushed by Lark |

## Prerequisites

1. A **Lark / Feishu** account with admin access to the [Open Platform](https://open.feishu.cn/) (Feishu) or [Lark Developer Portal](https://open.larksuite.com/) (International).
2. A **custom app** created in the developer console with the following permissions granted:
   - `helpdesk:ticket:readonly` — read service desk tickets (for pull mode)
   - `helpdesk:ticket` — manage service desk tickets (optional, for action support)
3. The app must be **published** (or in a sandbox/preview state) so that the API is accessible.

## Authentication

The Lark provider uses **OAuth 2.0 client credentials** (App ID + App Secret) to obtain a tenant access token automatically. Tokens are cached and refreshed before expiry.

| Field | Required | Description |
|-------|----------|-------------|
| `app_id` | ✅ | The App ID from the Lark Open Platform developer console |
| `app_secret` | ✅ | The App Secret from the developer console |
| `webhook_token` | ✗ | Verification token for incoming webhook events |
| `base_url` | ✗ | `https://open.feishu.cn` (default) or `https://open.larksuite.com` for international |

## Connecting Lark to Keep

### Step 1: Create a Lark App

1. Go to [https://open.feishu.cn/app](https://open.feishu.cn/app) (Feishu) or [https://open.larksuite.com/app](https://open.larksuite.com/app) (Lark).
2. Click **Create App** → **Custom App**.
3. Enter a name and description, then click **Create**.
4. Under **Credentials & Basic Info**, copy the **App ID** and **App Secret**.

### Step 2: Grant Permissions

1. In the app console, navigate to **Permissions & Scopes**.
2. Search for and enable `helpdesk:ticket:readonly`.
3. Click **Apply for Release** and publish the app to your organization.

### Step 3: Configure Webhook (for push mode)

1. In the app console, go to **Event Subscriptions**.
2. Set the **Request URL** to your Keep webhook endpoint:
   ```
   https://<your-keep-instance>/alerts/event/lark
   ```
3. Subscribe to the events you want:
   - `helpdesk.ticket.created_v1`
   - `helpdesk.ticket.updated_v1`
4. Copy the **Verification Token** and provide it as the `webhook_token` in Keep.

### Step 4: Add the Provider in Keep

1. Open Keep → **Providers** → **Connect Provider**.
2. Select **Lark / Feishu**.
3. Fill in the `app_id`, `app_secret`, and optionally `webhook_token` and `base_url`.
4. Click **Save**.

## Alert Schema

|------------|-------------|-------|
| `id` | `ticket_id` | Prefixed with `lark-ticket-` |
| `name` | `ticket.name` | Ticket title |
| `description` | `ticket.description` | Ticket body |
| `severity` | `ticket.urgency` | 1→Critical, 2→High, 3→Warning, 4→Info |
