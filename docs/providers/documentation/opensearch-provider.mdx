## Overview

The OpenSearch Provider connects Keep with [OpenSearch](https://opensearch.org/) in three modes:

- **Pull mode** — Keep polls the OpenSearch Alerting plugin (`/_plugins/_alerting/alerts`) to retrieve active monitors.
- **Query mode** — Execute DSL or SQL queries against any OpenSearch index.
- **Push mode (webhook)** — OpenSearch Alerting sends HTTP notifications to Keep's inbound webhook endpoint.

## Authentication

| Field | Required | Description |
|-------|----------|-------------|
| `url` | Yes | OpenSearch cluster URL, e.g. `https://opensearch.example.com:9200` |
| `username` | No* | HTTP Basic Auth username (e.g. `admin`) |
| `password` | No* | HTTP Basic Auth password |
| `api_key` | No* | API key in `id:api_key` base64-encoded format |
| `verify_ssl` | No | Verify SSL certificates (default: `true`). Set to `false` for self-signed certs. |

\* Either `username`+`password` or `api_key` must be provided.

## Push Mode: Configuring OpenSearch Alerting Webhook

1. In OpenSearch Dashboards, navigate to **Alerting** → **Notification channels**.
2. Click **Create channel** and choose **Custom webhook**.
3. Set the **URL** to your Keep webhook URL.
4. Add a custom header: `Authorization: Bearer {api_key}`.
5. Set the request body using the Mustache template:

```json
{
  "id": "{{ctx.monitor.id}}",
  "name": "{{ctx.monitor.name}}",
  "trigger_name": "{{ctx.trigger.name}}",
  "severity": "{{ctx.trigger.severity}}",
  "state": "{{ctx.results[0].hits.total.value}}",
  "message": "{{ctx.results[0]._source.message}}"
}
```

6. Click **Send test message** to verify, then assign the channel to a monitor trigger.
