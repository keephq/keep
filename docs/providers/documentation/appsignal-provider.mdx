## Overview

[AppSignal](https://www.appsignal.com/) is an application performance monitoring (APM) and error tracking platform that supports Ruby, Elixir, Node.js, Python, and other languages. It provides real-time insights into application errors, slow transactions, host metrics, and uptime checks.

## Connecting AppSignal to Keep

### Alert Integration via Webhook

To forward AppSignal notifications to Keep, configure a webhook integration in your AppSignal project settings:

### Step 1: Set Up the Webhook in AppSignal

1. Log in to your [AppSignal dashboard](https://appsignal.com).
2. Open your application and go to **App Settings** → **Notifications** → **Webhooks**.
3. Click **Add webhook**.
4. Enter your Keep webhook URL:

   ```
   {keep_webhook_api_url}
   ```

5. Add a custom header for authentication:
   - **Header name**: `X-API-KEY`
   - **Header value**: Your Keep API key

6. Select the event types you want to send to Keep (errors, performance, host, uptime, anomaly).
7. Click **Save** to activate the integration.

### Step 2: Verify the Integration

Trigger or simulate an alert in AppSignal and confirm it appears in your Keep alerts feed.

## Supported Event Types

AppSignal supports multiple webhook event types:

| Event Type | Description |
|---|---|
| `exception` | Unhandled application errors and exceptions |
| `performance` | Slow transaction alerts |
| `host` | Host metric alerts (CPU, memory, disk) |
| `uptime` | Uptime check failures |
| `anomaly` | Anomaly detection alerts |

## Alert Payload

AppSignal wraps webhook payloads in a type key. Keep automatically detects the event type and maps the fields accordingly:

| AppSignal Field | Keep Field |
|---|---|
| `id` | Alert ID |
| `exception` / `name` | Alert name |
| `message` / `description` | Description |
| `status` | Alert status (FIRING/RESOLVED) |
| `action` | Related action/endpoint |
| `url` | Alert dashboard URL |
