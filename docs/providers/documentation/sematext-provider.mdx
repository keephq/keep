## Overview

[Sematext Cloud](https://sematext.com/) is an observability platform providing infrastructure monitoring, log management, real user monitoring, and synthetic monitoring.

The Sematext provider connects Keep to Sematext using Sematext's **Custom Notification Hook** (webhook) feature. When an alert rule fires or recovers, Sematext sends a JSON payload to Keep's inbound webhook URL.

No API credentials are required — Keep acts as a pure webhook receiver.

## Connecting Sematext to Keep

1. Log in to [Sematext Cloud](https://apps.sematext.com/).
2. Navigate to the **App** you want to monitor (Monitoring, Logs, or Synthetics).
3. Go to **Alert Rules** and open an existing rule or create a new one.
4. In the **Notifications** section, click **Add Notification Hook**.
5. Select **Custom** (webhook) as the notification type.
6. Set the **URL** to the Keep webhook URL shown in your provider configuration.
7. Set **Method** to `POST` and **Content-Type** to `application/json`.
8. Leave the body blank — Sematext sends its default JSON payload automatically.
9. Click **Save** and save the rule.

Keep will now receive both firing and recovery notifications from this Sematext app.

## Alert Mapping

### Status Mapping

| Condition | Keep Status |
|-----------|-------------|
| `backToNormal: true` | Resolved |
| `ruleType: SCHEDULED` | Suppressed |
| `ruleType: RECOVERY` | Resolved |
| All other active alerts | Firing |

### Severity Mapping

Sematext alert priority maps to Keep severity:

| Sematext Priority | Keep Severity |
|-------------------|---------------|
| CRITICAL / `5` | Critical |
| ERROR / `4` | High |
| WARNING / `3` | Warning |
| INFO / `2` | Info |
| LOW / `1` | Low |

Recovery alerts (backToNormal=true) always have **Info** severity in Keep, regardless of the original priority.
