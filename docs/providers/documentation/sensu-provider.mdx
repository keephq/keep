## Overview

The Sensu Go Provider integrates Keep with [Sensu Go](https://sensu.io/), an open-source monitoring and observability pipeline for infrastructure, application, and cloud-native workloads.

Two integration modes are supported:

- **Pull mode** — Keep polls the Sensu Go REST API (`/api/core/v2/namespaces/{namespace}/events`) for non-OK events on a schedule.
- **Push mode (webhook)** — Sensu sends raw event payloads to Keep's inbound webhook endpoint via a Sensu HTTP handler.

## Authentication

Sensu Go supports two authentication methods. API key authentication is strongly recommended.

| Field | Required | Description |
|-------|----------|-------------|
| `sensu_host` | Yes | Sensu Go API base URL, e.g. `http://sensu.example.com:8080` |
| `api_key` | No* | Sensu Go API key (preferred) |
| `username` | No* | Sensu username for basic auth |
| `password` | No* | Sensu password for basic auth |
| `namespace` | No | Namespace to pull events from (default: `default`) |
| `pull_all_namespaces` | No | Set to `true` to pull events from all namespaces (cluster-wide) |

\* Either `api_key` **or** both `username` + `password` must be provided.

**To create an API key:**
```bash
sensuctl api-key grant admin
```
The output contains the API key UUID to use as `api_key`.

## Pull Mode

When pull mode is enabled, Keep calls:

- Single namespace: `GET /api/core/v2/namespaces/{namespace}/events`
- All namespaces: `GET /api/core/v2/events`

Only events with a non-zero `check.status` (i.e., WARNING, CRITICAL, or UNKNOWN) are surfaced as alerts in Keep. OK events (`status: 0`) are filtered out.

## Push Mode (Webhook)

To push Sensu events to Keep in real-time, configure a Sensu HTTP handler:

1. Create a Sensu handler definition (e.g. `keep-handler.yaml`):

```yaml
type: Handler
api_version: core/v2
metadata:
  name: keep
  namespace: default
spec:
  type: http
  timeout: 10
  http_handler:
    url: "{keep_webhook_api_url}"
    method: POST
    headers:
      Content-Type: application/json
```

2. Apply the handler:
```bash
sensuctl create -f keep-handler.yaml
```

3. Reference the handler in your check definitions:
```yaml
spec:
  handlers:
    - keep
```

Sensu will POST the raw event JSON to Keep's webhook endpoint when a check enters or leaves a non-OK state.

## Alert Mapping

### Severity Mapping

| Sensu Check Status | Keep Severity |
|--------------------|---------------|
| 0 (OK / Passing)   | Info          |
| 1 (Warning)        | Warning       |
| 2 (Critical)       | Critical      |
| ≥ 3 (Unknown)      | High          |

### Status Mapping

| Sensu Check State | Silenced | Keep Status |
|-------------------|----------|-------------|
| `passing`         | No       | Resolved    |
| `failing`         | No       | Firing      |
| `flapping`        | No       | Firing      |
| Any               | Yes      | Suppressed  |
| `0` (OK)          | No       | Resolved    |
