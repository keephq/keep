## Overview

[Lumigo](https://lumigo.io/) is a serverless observability platform that provides monitoring, debugging, and distributed tracing for AWS Lambda functions and other cloud-native applications. Lumigo helps engineering teams detect issues, reduce mean time to resolution (MTTR), and understand how their serverless infrastructure behaves in production.

## Connecting Lumigo to Keep

### Alert Integration via Webhook

To forward Lumigo alerts to Keep, configure a webhook in your Lumigo dashboard:

### Step 1: Set Up the Webhook in Lumigo

1. Log in to your [Lumigo dashboard](https://platform.lumigo.io).
2. Navigate to **Settings** → **Alerts**.
3. Click **Add Alert Channel** and select **Webhook**.
4. Paste your Keep webhook URL into the **Webhook URL** field:

   ```
   {keep_webhook_api_url}
   ```

5. Add a custom HTTP header for authentication:
   - **Header name**: `X-API-KEY`
   - **Header value**: Your Keep API key

6. Save the alert channel and configure which alerts should be forwarded to Keep.

### Step 2: Verify the Integration

After saving, trigger a test alert in Lumigo or wait for a real alert to fire. Check your Keep dashboard to confirm alerts are appearing correctly.

## Alert Payload

Lumigo sends webhook payloads in JSON format. Keep maps the following fields:

| Lumigo Field | Keep Field |
|---|---|
| `id` | Alert ID |
| `name` / `alertName` | Alert name |
| `description` / `message` | Description |
| `status` | Alert status (FIRING/RESOLVED) |
| `severity` | Alert severity |
| `service` / `functionName` | Service name |
| `region` | AWS region |
| `url` / `dashboardUrl` | Alert URL |
| `timestamp` / `triggeredAt` | Last received time |
