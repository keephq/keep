## Overview

[Betterstack](https://betterstack.com/) (formerly Better Uptime) is an uptime monitoring and incident management platform. It monitors your websites, APIs, and infrastructure and pages your team when something goes down.

The Keep Betterstack provider lets you:
- Pull active incidents from Betterstack into Keep via the REST API
- Receive real-time webhook notifications when monitors go down, get acknowledged, or recover

## Connecting Betterstack to Keep

### Prerequisites

You need a **Betterstack API token**. To get one:

1. Log in to [betterstack.com](https://betterstack.com/) and go to **Uptime**.
2. Navigate to **Settings** → **API** in the left sidebar.
3. Click **Create new API token**, give it a name, and copy the token.

### Webhook setup (recommended for real-time alerts)

1. In Betterstack Uptime, go to **On-call** → **Alert destinations**.
2. Click **New alert destination** and select **Webhook**.
3. Set the **Webhook URL** to your Keep webhook URL:
   ```
   https://api.keephq.dev/alerts/event/betterstack
   ```
4. Add a request header:
   - Key: `X-API-KEY`
   - Value: your Keep API key (found in Keep → Settings → API Keys)
5. Save the alert destination.
6. Go to **On-call** → **Policies** and add the webhook destination to an on-call policy.
7. Assign that policy to the monitors you want tracked in Keep.

### Pull mode (polling)

If you prefer polling over webhooks, provide your API token in the provider configuration. Keep will periodically pull active incidents from the Betterstack API.

## Alert fields

| Keep field | Betterstack source |
|---|---|
| `id` | Incident ID |
| `name` | Monitor name + "incident" |
| `description` | Monitor cause (e.g., "down") |
| `severity` | Mapped from cause: down → CRITICAL, ssl_expiring → WARNING |
| `status` | Mapped from status: started → FIRING, resolved → RESOLVED |
| `lastReceived` | `resolved_at` or `started_at` |
| `url` | Monitor URL |
