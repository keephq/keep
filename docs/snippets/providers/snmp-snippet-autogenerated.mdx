## Authentication
This provider requires authentication.
- **host**: Listen address for SNMP trap receiver (required: False, sensitive: False)
- **port**: UDP port to listen for SNMP traps (required: False, sensitive: False)
- **community_string**: SNMP community string for SNMPv1/v2c (required: False, sensitive: True)
- **version**: SNMP version — 1, 2c, or 3 (required: False, sensitive: False)
- **username**: SNMPv3 username (required: False, sensitive: False)
- **auth_key**: SNMPv3 authentication key (required: False, sensitive: True)
- **auth_protocol**: SNMPv3 auth protocol — MD5 or SHA (required: False, sensitive: False)
- **priv_key**: SNMPv3 privacy (encryption) key (required: False, sensitive: True)
- **priv_protocol**: SNMPv3 privacy protocol — DES or AES (required: False, sensitive: False)
- **oids_mapping**: JSON map of OID prefixes to alert settings (required: False, sensitive: False)
- **poll_enabled**: Enable periodic SNMP polling of target devices (required: False, sensitive: False)
- **poll_targets**: JSON list of polling targets (required: False, sensitive: False)
- **poll_interval**: Polling interval in seconds (required: False, sensitive: False)



## In workflows

This provider can't be used as a "step" or "action" in workflows. It operates as a listener that receives SNMP traps and converts them into Keep alerts.



## Connecting via Webhook
This provider receives SNMP traps via Keep's built-in trap listener.

To configure your network devices to send SNMP traps to Keep:

    1. Install the SNMP provider in Keep and note the configured host and port (default: 0.0.0.0:162).
    2. For SNMPv1/v2c, set the community string to match your device configuration.
    3. For SNMPv3, configure the username, auth_key, auth_protocol, priv_key, and priv_protocol to match your device's SNMPv3 credentials.
    4. On your network devices (routers, switches, servers), configure the SNMP trap destination to point to the Keep instance's IP and port.
    5. Optionally, configure oids_mapping to map specific OID prefixes to custom alert names and severities.
    6. Optionally, enable poll_enabled and configure poll_targets to periodically query SNMP OIDs from devices.
