## Authentication
This provider uses optional authentication.
- **host**: Listen address for the built-in SNMP trap receiver (required: False, sensitive: False)
- **port**: UDP port for SNMP traps — default 162, use 1620+ for non-root (required: False, sensitive: False)
- **community_string**: SNMP community string for SNMPv1/v2c (required: False, sensitive: True)
- **version**: SNMP version: '1', '2c', or '3' (required: False, sensitive: False)
- **username**: SNMPv3 username (required for v3) (required: False, sensitive: False)
- **auth_key**: SNMPv3 authentication passphrase (required: False, sensitive: True)
- **auth_protocol**: SNMPv3 auth protocol: 'MD5' or 'SHA' (required: False, sensitive: False)
- **priv_key**: SNMPv3 privacy (encryption) passphrase (required: False, sensitive: True)
- **priv_protocol**: SNMPv3 privacy protocol: 'DES' or 'AES' (required: False, sensitive: False)
- **oids_mapping**: JSON map of OID prefixes → custom severity/name overrides (required: False, sensitive: False)
- **poll_enabled**: Enable periodic SNMP polling (required: False, sensitive: False)
- **poll_targets**: JSON list of polling targets (required: False, sensitive: False)
- **poll_interval**: Polling interval in seconds (required: False, sensitive: False)



## In workflows

This provider can be used as a step or action in workflows to query SNMP OIDs from devices.



## Connecting via Webhook
This provider receives SNMP traps via Keep's built-in UDP trap listener or via HTTP webhook from snmptrapd.

To configure your network devices to send SNMP traps to Keep:

    1. Install the SNMP provider in Keep and note the configured host and port (default: 0.0.0.0:162).
    2. For SNMPv1/v2c, set the community string to match your device configuration.
    3. For SNMPv3, configure the username, auth_key, auth_protocol, priv_key, and priv_protocol.
    4. Configure your network devices to send traps to the Keep instance IP and port.
    5. Alternatively, use snmptrapd to forward traps as JSON to Keep's webhook endpoint.
