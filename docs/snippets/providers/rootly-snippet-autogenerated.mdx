## Authentication
This provider requires authentication.
- **api_key**: Rootly API Key (Global, Team, or Personal) (required: True, sensitive: True)
- **api_url**: Rootly API base URL (required: False, sensitive: False)
- **pull_incidents**: Also pull incidents (in addition to alerts) (required: False, sensitive: False)

Certain scopes may be required to perform specific actions or queries via the provider. Below is a summary of relevant scopes and their use cases:
- **authenticated**: User is authenticated with Rootly API (mandatory) 
- **read_alerts**: User can read alerts from Rootly (mandatory) 
- **read_incidents**: User can read incidents from Rootly

## Connecting via Webhook (omnidirectional)

Receive Rootly alerts and incident events


To send Rootly events to Keep, configure a webhook in Rootly:

1. Go to **Rootly** → **Settings** → **Integrations** → **Webhooks**
2. Click **Add Webhook**
3. Set the **Endpoint URL** to: `KEEP_BACKEND_URL/alerts/event/rootly`
4. Under **Events**, select the events you want to forward:
   - `alert.created`, `alert.updated`, `alert.resolved`
   - `incident.created`, `incident.updated`, `incident.mitigated`, `incident.resolved`, `incident.cancelled`
5. Add a header `X-API-KEY: {api_key}` under **Headers**
6. Click **Save**

Keep will now receive real-time alerts and incident updates from Rootly.
